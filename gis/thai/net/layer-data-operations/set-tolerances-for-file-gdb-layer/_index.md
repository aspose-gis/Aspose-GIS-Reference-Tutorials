# วิธีสร้างชุดข้อมูล GDB และตั้งค่าความทนทานสำหรับเลเยอร์

## บทนำ
หากคุณต้องการ **สร้างชุดข้อมูลไฟล์ GDB** และควบคุมความแม่นยำของมัน คุณมาถูกที่แล้ว ในบทแนะนำนี้เราจะพาคุณผ่านกระบวนการทั้งหมด—เริ่มจากการตั้งค่าโครงการ .NET ของคุณ, การสร้าง File Geodatabase (GDB) dataset, แล้วจึงนำความทนทาน XY, Z, และ M ไปใช้กับเลเยอร์ใหม่ เมื่อเสร็จคุณจะมีชุดข้อมูลพร้อมใช้งานที่ทำงานได้อย่างราบรื่นกับเครื่องมือ ArcGIS และแอปพลิเคชัน GIS อื่น ๆ คู่มือนี้แสดงให้คุณเห็น **วิธีสร้าง gdb** ไฟล์โดยโปรแกรมเมติก เพื่อให้คุณสามารถทำอัตโนมัติขั้นตอนข้อมูลโดยไม่ต้องทำด้วยตนเอง

## คำตอบสั้น
- **“create file GDB dataset” หมายความว่าอย่างไร?** มันสร้างคอนเทนเนอร์ File Geodatabase ใหม่บนดิสก์ที่สามารถเก็บหลายเลเยอร์ GIS ได้  
- **ทำไมต้องตั้งค่าความทนทาน?** ความทนทานกำหนดความแม่นยำสำหรับการดำเนินการเรขาคณิต เพื่อป้องกันข้อผิดพลาดจากการปัดเศษในการวิเคราะห์เชิงพื้นที่  
- **คลาส Aspose.GIS ใดที่ใช้?** `Dataset.Create` ร่วมกับ `FileGdbOptions`  
- **ฉันต้องการไลเซนส์สำหรับการพัฒนาหรือไม่?** ไลเซนส์ชั่วคราวเพียงพอสำหรับการทดสอบ; ไลเซนส์เต็มจำเป็นสำหรับการใช้งานจริง  
- **เวอร์ชัน .NET ที่รองรับคืออะไร?** .NET Framework 4.5+, .NET Core 3.1+, .NET 5/6/7  

## File GDB Dataset คืออะไร?
File Geodatabase (GDB) คือที่เก็บข้อมูลแบบโฟลเดอร์ที่บรรจุเลเยอร์ GIS, ตาราง, และความสัมพันธ์ต่าง ๆ โดยใช้ Aspose.GIS คุณสามารถ **สร้างไฟล์ GDB dataset** ด้วยโปรแกรมโดยไม่ต้องติดตั้ง ArcGIS ทำให้เหมาะสำหรับการทำงานอัตโนมัติหรือแอปพลิเคชันที่กำหนดเอง

## ทำไมต้องตั้งค่าความทนทานสำหรับเลเยอร์?
การตั้งค่าความทนทานทำให้การคำนวณเรขาคณิต (เช่น การตัดกัน, การบัฟเฟอร์, หรือการสแนป) มีความแม่นยำตามที่คุณต้องการ สิ่งนี้สำคัญอย่างยิ่งเมื่อทำงานกับข้อมูลความละเอียดสูงหรือเมื่อส่งออกไปยังแพลตฟอร์ม GIS อื่น ๆ ที่คาดหวังค่าความทนทานเฉพาะ กล่าวอีกอย่างคือคุณกำลัง **ตั้งค่าความแม่นยำของเลเยอร์** เพื่อหลีกเลี่ยงข้อผิดพลาดของเรขาคณิตที่ไม่คาดคิด

## นำเข้าเนมสเปซ
ในแอปพลิเคชัน .NET ของคุณ ให้รวมเนมสเปซต่อไปนี้เพื่อใช้ประโยชน์จากฟังก์ชันของ Aspose.GIS:

```csharp
using Aspose.Gis;
using Aspose.Gis.Formats.FileGdb;
using Aspose.Gis.Geometries;
using Aspose.Gis.SpatialReferencing;
using System;
using System.Text;
```

เมื่อมีเนมสเปซแล้ว เราสามารถเริ่มสร้างชุดข้อมูลได้

## วิธีสร้างชุดข้อมูล GDB?
ด้านล่างเป็นคู่มือขั้นตอนต่อขั้นตอนที่พาคุณผ่านการสร้างชุดข้อมูลและการกำหนดค่าความทนทาน

### ขั้นตอนที่ 1: กำหนดไดเรกทอรีเอกสารของคุณ
ก่อนอื่น ให้ชี้โค้ดไปยังโฟลเดอร์ที่คุณต้องการสร้าง File GDB:

```csharp
string dataDir = "Your Document Directory";
```

> **เคล็ดลับ:** ใช้ `Path.Combine` หากคุณต้องการสร้างเส้นทางแบบไม่ขึ้นกับแพลตฟอร์ม

### ขั้นตอนที่ 2: สร้าง File GDB Dataset
ตอนนี้เราจริง ๆ **สร้างไฟล์ GDB dataset** บนดิสก์ เมธอด `Dataset.Create` รับพาธเต็มและประเภทไดรเวอร์ (`Drivers.FileGdb`).

```csharp
var path = dataDir + "TolerancesForFileGdbLayer_out.gdb";
using (var dataset = Dataset.Create(path, Drivers.FileGdb))
{
```

> บล็อก `using` ทำให้แน่ใจว่าชุดข้อมูลถูกปิดอย่างถูกต้องและบันทึกลงดิสก์เมื่อคุณเสร็จสิ้น

### ขั้นตอนที่ 3: ตั้งค่าความทนทานโดยใช้ `FileGdbOptions`
ก่อนสร้างเลเยอร์ ให้กำหนดค่าความทนทานที่คุณต้องการ `FileGdbOptions` ให้คุณระบุความทนทาน XY, Z, และ M — นี่คืออ็อบเจกต์ **file gdb options** ที่ควบคุมความแม่นยำ

```csharp
var options = new FileGdbOptions
{
    XYTolerance = 0.001,
    ZTolerance = 0.1,
    MTolerance = 0.1,
};
```

ค่าต่าง ๆ เหล่านี้เป็นค่ามาตรฐานสำหรับข้อมูลวิศวกรรมความแม่นยำสูง แต่คุณสามารถปรับให้เหมาะกับโครงการของคุณได้

### ขั้นตอนที่ 4: สร้างเลเยอร์ GIS ด้วยความทนทานที่กำหนด
สุดท้าย สร้างเลเยอร์ใหม่ภายในชุดข้อมูลโดยส่งอ็อบเจกต์ options ที่เราตั้งค่าไว้ ขั้นตอนนี้แสดง **วิธีตั้งค่าความทนทาน** พร้อมกับ **การสร้างเลเยอร์ GIS**.

```csharp
using (var layer = dataset.CreateLayer("layer_name", options))
{
    // The layer is created with the provided tolerances, ready for use in ArcGIS features/tools.
}
```

เมื่อบล็อก `using` สิ้นสุด เลเยอร์จะถูกบันทึกพร้อมกับความทนทานที่คุณกำหนด

## ปัญหาทั่วไปและวิธีแก้
| ปัญหา | สาเหตุ | วิธีแก้ |
|-------|--------|----------|

| **ค่าความทนทานไม่ถูกต้อง** | ความทนทานต้องเป็นตัวเลขที่ไม่เป็นลบ | ใช้ค่าบวก; หลีกเลี่ยงศูนย์ เว้นแต่คุณต้องการไม่มีความทนทานโดยเจตนา |
