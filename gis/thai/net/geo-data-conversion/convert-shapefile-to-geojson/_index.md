# แปลง Shapefile เป็น GeoJSON

## บทนำ
ในระบบสารสนเทศภูมิศาสตร์ (GIS) สมัยใหม่ **การทำงานร่วมกันของข้อมูลเชิงพื้นที่** เป็นกุญแจสำคัญในการเปิดประโยชน์การวิเคราะห์เชิงพื้นที่ที่ทรงพลัง หนึ่งในงานแปลงที่พบบ่อยที่สุดคือ **การแปลง shapefile เป็น geojson** ซึ่งทำให้การแลกเปลี่ยนข้อมูลที่มีน้ำหนักเบากับแผนที่เว็บ แอปมือถือ และบริการคลาวด์เป็นไปได้อย่างราบรื่น บทแนะนำนี้จะพาคุณผ่านกระบวนการทั้งหมดโดยใช้ไลบรารี **Aspose.GIS .NET** เพื่อให้คุณสามารถรวมการแปลงนี้เข้าไปในแอปพลิเคชัน C# ของคุณได้โดยตรง

## คำตอบด่วน
- **ไลบรารีใดจัดการการแปลง?** Aspose.GIS for .NET  
- **การดำเนินการใช้เวลานานเท่าไหร่?** ปกติไม่เกิน 10 นาทีสำหรับไฟล์เดียว  
- **ฉันต้องการไลเซนส์หรือไม่?** เวอร์ชันทดลองฟรีใช้ได้สำหรับการพัฒนา; ต้องมีไลเซนส์สำหรับการใช้งานจริง  
- **เวอร์ชัน .NET ที่รองรับ?** .NET Framework 4.5+, .NET Core 3.1+, .NET 5/6/7  
- **ฉันสามารถแปลงหลายไฟล์ได้หรือไม่?** ได้ – เพียงทำลูปเรียก `VectorLayer.Convert`  

## อะไรคือ “แปลง shapefile เป็น geojson”?
การแปลง Shapefile (ชุดไฟล์ `.shp`, `.shx`, `.dbf`) ให้เป็น GeoJSON จะทำให้ข้อมูลอยู่ในรูปแบบ JSON‑single ที่อ่านง่าย แก้ไขได้ง่าย และสามารถแสดงผลในเว็บเบราว์เซอร์ได้อย่างรวดเร็ว GeoJSON เหมาะอย่างยิ่งกับไลบรารีการทำแผนที่ JavaScript เช่น Leaflet หรือ Mapbox

## ทำไมต้องใช้ Aspose.GIS for .NET สำหรับการแปลงรูปแบบข้อมูล GIS?
- **API ครบวงจร** – รองรับรูปแบบหลายสิบรูปแบบ (TIFF, KML, GML ฯลฯ) โดยไม่ต้องใช้เครื่องมือภายนอก  
- **การแปลงไม่มีการพึ่งพา** – ไม่ต้องติดตั้ง GDAL หรือไลบรารีเนทีฟอื่น ๆ  
- **ประสิทธิภาพสูง** – ปรับให้ทำงานได้ดีกับชุดข้อมูลขนาดใหญ่และการประมวลผลเป็นชุด  
- **การปรับแต่งที่หลากหลาย** – สามารถระบุระบบพิกัด, ตัวกรองแอตทริบิวต์, และอื่น ๆ ได้  

## ข้อกำหนดเบื้องต้น
ก่อนเริ่มทำงาน โปรดตรวจสอบว่าคุณมีสิ่งต่อไปนี้:

1. **Aspose.GIS for .NET ติดตั้งแล้ว** – ทำตามคำแนะนำใน [เอกสาร Aspose.GIS for .NET อย่างเป็นทางการ](https://reference.aspose.com/gis/net/) เพื่อเพิ่มแพคเกจ NuGet ลงในโปรเจกต์ของคุณ  
2. **Shapefile ต้นทาง** – ดาวน์โหลดจากพอร์ทัลข้อมูลเปิด, หน่วยงานรัฐบาล, หรือสร้างด้วย QGIS/ArcGIS  
3. **ความรู้พื้นฐาน C#** – โค้ดตัวอย่างใช้ไวยากรณ์ C# และแนวปฏิบัติของ .NET  

## นำเข้า Namespaces
ก่อนอื่นให้นำเข้า namespaces ที่ให้คุณเข้าถึงคลาสของ Aspose.GIS และยูทิลิตี้มาตรฐานของ .NET:

```csharp
using Aspose.Gis;
using System;
using System.Collections.Generic;
using System.Linq;
using System.Text;
using System.Threading.Tasks;
```

## คู่มือขั้นตอนต่อขั้นตอน

### ขั้นตอนที่ 1: กำหนดเส้นทางอินพุตและเอาต์พุต
กำหนดโฟลเดอร์ที่มี Shapefile ของคุณและตำแหน่งปลายทางสำหรับไฟล์ GeoJSON ปรับเส้นทางให้ตรงกับสภาพแวดล้อมของคุณ

```csharp
string dataDir = "Your Document Directory";
string shapefilePath = dataDir + "InputShapeFile.shp";
string jsonPath = dataDir + "output_out.json";
```

> **เคล็ดลับ:** ใช้ `Path.Combine(dataDir, "InputShapeFile.shp")` เพื่อสร้างเส้นทางที่ทำงานได้บนทุกแพลตฟอร์ม

### ขั้นตอนที่ 2: ดำเนินการแปลง
เรียกเมธอดสถิต `VectorLayer.Convert` เมธอดนี้จะอ่าน Shapefile, แปลงข้อมูล, และเขียนไฟล์ GeoJSON

```csharp
VectorLayer.Convert(shapefilePath, Drivers.Shapefile, jsonPath, Drivers.GeoJson);
```

หลังจากทำงานเสร็จ `output_out.json` จะมี **GeoJSON FeatureCollection** ที่ถูกต้องซึ่งคุณสามารถโหลดเข้าสู่ตัวดูแผนที่เว็บใดก็ได้

## ปัญหาทั่วไปและวิธีแก้
| ปัญหา | สาเหตุ | วิธีแก้ |
|-------|--------|---------|
| **ไฟล์ไม่พบ** | `dataDir` ไม่ถูกต้องหรือไฟล์ `.shp` ขาดหาย | ตรวจสอบเส้นทางและให้แน่ใจว่ามีส่วนประกอบของ Shapefile ทั้งสาม (`.shp`, `.shx`, `.dbf`) อยู่ครบ |
| **ระบบพิกัดไม่ตรงกัน** | Shapefile ต้นทางใช้โปรเจกชันที่ผู้รับไม่รู้จัก | ใช้ `VectorLayer.Open(...).CoordinateSystem` เพื่อทำการรีโปรเจกต์ก่อนแปลง |
| **ไฟล์ขนาดใหญ่ทำให้หน่วยความจำอัด** | โหลดชุดข้อมูลทั้งหมดเข้าสู่หน่วยความจำ | ประมวลผลฟีเจอร์เป็นชิ้นส่วนหรือใช้ `VectorLayer.Stream` สำหรับการแปลงแบบสตรีมมิ่ง |

## คำถามที่พบบ่อย

**ถาม: ฉันสามารถแปลงหลาย Shapefile เป็น GeoJSON พร้อมกันได้หรือไม่โดยใช้ Aspose.GIS for .NET?**  
ตอบ: ได้. ให้วางโค้ดการแปลงไว้ในลูป `foreach` ที่วนผ่านไฟล์ `.shp` ทุกไฟล์ในไดเรกทอรี

**ถาม: Aspose.GIS for .NET รองรับทุกเวอร์ชันของ .NET Framework หรือไม่?**  
ตอบ: รองรับ .NET Framework 4.5 ขึ้นไป รวมถึง .NET Core 3.1+ และ .NET 5/6/7

**ถาม: Aspose.GIS for .NET มีการสนับสนุนรูปแบบเชิงพื้นที่อื่น ๆ นอกจาก Shapefile และ GeoJSON หรือไม่?**  
ตอบ: แน่นอน. ไลบรารีรองรับรูปแบบเช่น GeoTIFF, KML, GML, CSV และอีกหลายรูปแบบ

**ถาม: ฉันสามารถปรับแต่งกระบวนการแปลงได้หรือไม่ เช่น ระบุระบบพิกัดหรือแมปแอตทริบิวต์?**  
ตอบ: ได้. API มี overloads และ property ให้ตั้งค่าระบบพิกัดเป้าหมาย, ตัวกรองแอตทริบิวต์, และการแก้ไขเรขาคณิตของฟีเจอร์ระหว่างการแปลง

**ถาม: มีเวอร์ชันทดลองของ Aspose.GIS for .NET หรือไม่?**  
ตอบ: มี, คุณสามารถดาวน์โหลดเวอร์ชันทดลองฟรีจาก [เว็บไซต์ Aspose](https://releases.aspose.com/)
