## Εισαγωγή Χώρων Ονομάτων
Για να ξεκινήσετε, πρέπει να εισάγετε τους απαραίτητους χώρους ονομάτων στο έργο σας. Αυτοί οι χώροι ονομάτων παρέχουν τις βασικές κλάσεις και μεθόδους για αλληλεπίδραση με το Aspose.GIS για .NET.

```csharp
using Aspose.Gis;
using Aspose.Gis.Formats.FileGdb;
using Aspose.Gis.Geometries;
using System;
using Aspose.Gis.SpatialReferencing;
```

## Βήμα 1: Προσθήκη σημειακού χαρακτηριστικού και καθορισμός ονομάτων πεδίων Object ID και Geometry
Σε αυτό το βήμα, θα μάθετε πώς να ρυθμίσετε τα ονόματα πεδίων Object ID και Geometry για τα GIS δεδομένα σας. Αυτό είναι κρίσιμο για αποδοτική διαχείριση δεδομένων και για τη δυνατότητα **ανάγνωσης object id** αργότερα.

### Βήμα 1.1: Ορισμός Καταλόγου Εγγράφων
Ξεκινήστε ορίζοντας τη διαδρομή προς τον κατάλογο εγγράφων σας:

```csharp
string dataDir = "Your Document Directory";
```

### Βήμα 1.2: Δημιουργία GeoDatabase και Ορισμός Επιλογών
Δημιουργήστε ένα GeoDatabase με τα επιθυμητά **ονόματα πεδίων** για Object ID και Geometry:

```csharp
var path = dataDir + "NamesOfObjectIdAndGeometryFields_out.gdb";
using (var dataset = Dataset.Create(path, Drivers.FileGdb))
{
    var options = new FileGdbOptions
    {
        ObjectIdFieldName = "OID",         // Specify the Object ID field name
        GeometryFieldName = "POINT",       // Specify the Geometry field name
    };
```

### Βήμα 1.3: Δημιουργία και Προσθήκη Layer
Δημιουργήστε ένα layer μέσα στο GeoDatabase και προσθέστε ένα χαρακτηριστικό που αντιπροσωπεύει γεωμετρία σημείου:

```csharp
using (var layer = dataset.CreateLayer("layer_name", options, SpatialReferenceSystem.Wgs84))
{
    var feature = layer.ConstructFeature();
    feature.Geometry = new Point(12.32, 34.21);  // Specify the geometry (in this case, a point)
    layer.Add(feature);
}
```

### Βήμα 1.4: Άνοιγμα και Ανάκτηση Δεδομένων από το Layer
Ανοίξτε το layer και ανακτήστε το χαρακτηριστικό που προσθέσατε μόλις τώρα. Αυτό δείχνει πώς να **διαβάσετε object id** από το σύνολο δεδομένων:

```csharp
using (var layer = dataset.OpenLayer("layer_name"))
{
    var feature = layer[0];
    Console.WriteLine(feature.GetValue<int>("OID")); // Output: 1
}
```

## Γιατί να ορίσετε προσαρμοσμένα ονόματα πεδίων;
Η προσαρμογή των ονομάτων πεδίων Object ID και Geometry σας δίνει ευελιξία κατά την ενσωμάτωση με υπάρχουσες βάσεις δεδομένων ή όταν τηρείτε τις συμβάσεις ονοματοδοσίας που απαιτούνται από εφαρμογές downstream. Επίσης, κάνει τα δεδομένα πιο αυτο‑περιγραφικά, κάτι που απλοποιεί τη συντήρηση και τη συνεργασία.

## Συνηθισμένα προβλήματα και αντιμετώπιση
- **Λείπει ο φάκελος** – Βεβαιωθείτε ότι το `dataDir` δείχνει σε έναν υπάρχοντα φάκελο· διαφορετικά, το `Dataset.Create` θα ρίξει εξαίρεση.
- **Σύγκρουση ονομάτων πεδίων** – Εάν υπάρχει ήδη ένα πεδίο με το ίδιο όνομα στη γεωβάση, η δημιουργία θα αποτύχει. Επιλέξτε μοναδικά ονόματα.
- **Ασυμφωνία χωρικού συστήματος αναφοράς** – Πάντα ταιριάξτε το σύστημα χωρικής αναφοράς (π.χ., WGS84) με τα δεδομένα που σκοπεύετε να αποθηκεύσετε.

## Επιπλέον Συχνές Ερωτήσεις

**Ε: Μπορώ να αλλάξω το πεδίο Object ID μετά τη δημιουργία του layer;**  
Α: Όχι. Το όνομα του πεδίου ορίζεται κατά τη δημιουργία του layer· πρέπει να δημιουργήσετε ξανά το layer με ένα νέο αντικείμενο `FileGdbOptions` για να το αλλάξετε.

**Ε: Πώς μπορώ να ανακτήσω άλλες τιμές χαρακτηριστικών εκτός του Object ID;**  
Α: Χρησιμοποιήστε `feature.GetValue<T>("FieldName")` όπου `FieldName` είναι το όνομα του χαρακτηριστικού που θέλετε να διαβάσετε.

**Ε: Είναι δυνατόν να προσθέσω πολλαπλά σημειακά χαρακτηριστικά σε μία παρτίδα;**  
Α: Ναι. Επανάλαβε τα δεδομένα σας, δημιουργήστε ένα χαρακτηριστικό για κάθε σημείο, ορίστε τη γεωμετρία του, και καλέστε `layer.Add(feature)` μέσα στο ίδιο μπλοκ `using`.

**Ε: Υποστηρίζει το Aspose.GIS άλλους τύπους γεωμετρίας;**  
Α: Απόλυτα. Μπορείτε να εργαστείτε με `LineString`, `Polygon`, `MultiPoint` και άλλα δημιουργώντας τα κατάλληλα αντικείμενα γεωμετρίας.

**Ε: Τι συμβαίνει αν προσπαθήσω να διαβάσω ένα Object ID που δεν υπάρχει;**  
Α: Η πρόσβαση σε δείκτη εκτός εύρους (π.χ., `layer[10]` όταν υπάρχει μόνο ένα χαρακτηριστικό) θα προκαλέσει `IndexOutOfRangeException`. Πάντα ελέγξτε πρώτα το `layer.Count`.
