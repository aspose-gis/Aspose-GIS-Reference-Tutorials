# Μετατροπή Γεωμετρίας σε Επιλογές Παραλλαγής WKT χρησιμοποιώντας το Aspose.GIS

## Εισαγωγή
Σε σύγχρονες εφαρμογές .NET, η **μετατροπή γεωμετρίας σε WKT** είναι ένα συνηθισμένο βήμα όταν χρειάζεται να ανταλλάξετε χωρικά δεδομένα με άλλα συστήματα ή να τα αποθηκεύσετε σε μορφή κειμένου. Το Aspose.GIS για .NET κάνει αυτή τη μετατροπή απλή, ενώ σας δίνει πλήρη έλεγχο πάνω στην παραλλαγή WKT, τη μορφή αριθμών και την ακρίβεια των δεκαδικών. Σε αυτό το tutorial θα μάθετε πώς να μετατρέπετε γεωμετρία σε WKT, να επιλέγετε τη σωστή παραλλαγή και να ρυθμίζετε την έξοδο ώστε να καλύπτει ακριβώς τις απαιτήσεις του έργου σας.

## Γρήγορες Απαντήσεις
- **Τι σημαίνει η “μετατροπή γεωμετρίας σε WKT”;** Μετατρέπει γεωμετρικά αντικείμενα (σημεία, γραμμές, πολύγωνα) σε αναπαράσταση Well‑Known Text.  
- **Ποιες παραλλαγές WKT υποστηρίζονται;** `Iso`, `SimpleFeatureAccessOutdated` και `ExtendedPostGis`.  
- **Πώς μπορώ να ελέγξω την ακρίβεια των δεκαδικών;** Χρησιμοποιήστε τις επιλογές `NumericFormat` όπως `General`, `RoundTrip` ή `Flat`.  
- **Χρειάζομαι άδεια για παραγωγή;** Ναι, απαιτείται εμπορική άδεια για χρήση εκτός δοκιμής.  
- **Ποιες εκδόσεις .NET είναι συμβατές;** .NET Framework 4.0+ και .NET 5/6/7.

## Τι είναι η “μετατροπή γεωμετρίας σε WKT”;
Η μετατροπή γεωμετρίας σε WKT παράγει μια αναγνώσιμη από άνθρωπο συμβολοσειρά που περιγράφει χωρικά αντικείμενα. Αυτή η μορφή είναι ευρέως αποδεκτή από εργαλεία GIS, βάσεις δεδομένων και web services, καθιστώντας την ιδανική για ανταλλαγή δεδομένων.

## Γιατί να χρησιμοποιήσετε το Aspose.GIS για αυτή τη μετατροπή;
- **Έλεγχος ακρίβειας** – Επιλέξτε πόσες δεκαδικές θέσεις θα εκτυπώνονται.  
- **Ευελιξία παραλλαγής** – Ταιριάξτε ακριβώς το διάλεκτο WKT που απαιτεί το σύστημα προορισμού.  
- **Χωρίς εξωτερικές εξαρτήσεις** – Καθαρή βιβλιοθήκη .NET, χωρίς εγγενή binaries.

2. **Περιβάλλον Ανάπτυξης** – Οποιαδήποτε πρόσφατη έκδοση του Visual Studio ή VS Code με .NET SDK.

## Εισαγωγή Χώρων Ονομάτων
Πρώτα, φέρτε τα απαιτούμενα namespaces στο scope:

```csharp
using Aspose.Gis.Geometries;
using Aspose.Gis.SpatialReferencing;
using System;
using System.Collections.Generic;
using System.Linq;
using System.Text;
using System.Threading.Tasks;
using Aspose.Gis;
```

## Βήμα 1: Δημιουργία Αντικειμένου Point
Δημιουργήστε ένα `Point` που θα χρησιμοποιήσετε αργότερα για **μετατροπή γεωμετρίας σε WKT**. Το σημείο περιλαμβάνει γεωγραφικό πλάτος, μήκος και προαιρετική τιμή μέτρησης (M):

```csharp
Point point = new Point(23.5732, 25.3421) { M = 40.3 };
```

## Βήμα 2: Ορισμός Συστήματος Αναφοράς Χώρου (SRS)
Αντιστοιχίστε ένα σύστημα αναφοράς χώρου ώστε η έξοδος WKT να γνωρίζει σε ποιο σύστημα συντεταγμένων αναφέρεται. Εδώ χρησιμοποιούμε το κοινό σύστημα WGS84:

```csharp
point.SpatialReferenceSystem = SpatialReferenceSystem.Wgs84;
```

## Βήμα 3: Καθορισμός Παραλλαγής WKT
Επιλέξτε τη σωστή παραλλαγή WKT όταν **μετατρέπετε γεωμετρία σε WKT**. Κάθε παραλλαγή μορφοποιεί την έξοδο ελαφρώς διαφορετικά:

```csharp
Console.WriteLine(point.AsText(WktVariant.Iso)); // POINT M (23.5732, 25.3421, 40.3)
Console.WriteLine(point.AsText(WktVariant.SimpleFeatureAccessOutdated)); // POINT (23.5732, 25.3421)
Console.WriteLine(point.AsText(WktVariant.ExtendedPostGis)); // SRID=4326;POINTM (23.5732, 25.3421, 40.3)
```

## Βήμα 4: Έλεγχος Μορφής Αριθμών (Ορισμός Μορφής Αριθμών & Προσαρμογή Ακρίβειας Δεκαδικών)
Ρυθμίστε την αριθμητική αναπαράσταση των συντεταγμένων. Η κλάση `NumericFormat` σας επιτρέπει να **ορίσετε μορφή αριθμών** και να **προσαρμόσετε την ακρίβεια των δεκαδικών** σύμφωνα με τις ανάγκες σας:

```csharp
Console.WriteLine("G17  : " + point.AsText(WktVariant.Iso, NumericFormat.General(17))); // POINT M (23.5732 25.342099999999999 40.299999999999997)
Console.WriteLine("R    : " + point.AsText(WktVariant.Iso, NumericFormat.RoundTrip)); // POINT M (23.5732 25.3421 40.3)
Console.WriteLine("G3   : " + point.AsText(WktVariant.Iso, NumericFormat.General(3))); // POINT M (23.6 25.3 40.3)
Console.WriteLine("Flat3: " + point.AsText(WktVariant.Iso, NumericFormat.Flat(3))); // POINT M (23.573 25.342 40.3)
```

## Κοινά Προβλήματα & Συμβουλές
- **Λείπει η τιμή M** – Αν δεν χρειάζεστε μέτρηση, παραλείψτε την ιδιότητα `M`; η παραλλαγή ISO θα την αφαιρέσει αυτόματα.  
- **Απρόσμενη ακρίβεια** – Ελέγξτε ότι χρησιμοποιείτε το σωστό `NumericFormat`; το `General` διατηρεί πλήρη ακρίβεια double, ενώ το `Flat` στρογγυλοποιεί στον καθορισμένο αριθμό δεκαδικών.  
- **SRID δεν εμφανίζεται** – Μόνο η παραλλαγή `ExtendedPostGis` προσθέτει το πρόθεμα `SRID=` στην έξοδο. Επιλέξτε αυτή τη παραλλαγή όταν το σύστημα προορισμού απαιτεί SRID.
