# إنشاء مجموعة بيانات File GDB وتعيين التحملات للطبقة

## مقدمة
إذا كنت بحاجة إلى **إنشاء مجموعة بيانات File GDB** والتحكم في دقتها، فأنت في المكان المناسب. في هذا البرنامج التعليمي سنستعرض العملية بالكامل—بدءًا من إعداد مشروع .NET الخاص بك، وإنشاء مجموعة بيانات File Geodatabase (GDB)، ثم تطبيق تحملات XY و Z و M على طبقة جديدة. في النهاية ستحصل على مجموعة بيانات جاهزة للاستخدام تعمل بسلاسة مع أدوات ArcGIS وتطبيقات GIS الأخرى.

## إجابات سريعة
- **ماذا يعني “إنشاء مجموعة بيانات File GDB”؟** ينشئ حاوية File Geodatabase جديدة على القرص يمكنها احتواء طبقات GIS متعددة.  
- **لماذا نعيّن التحملات؟** تحدد التحملات الدقة لعمليات الهندسة، مما يمنع أخطاء التقريب في التحليل المكاني.  
- **أي فئة Aspose.GIS تُستخدم؟** `Dataset.Create` مع `FileGdbOptions`.  
- **هل أحتاج إلى ترخيص للتطوير؟** ترخيص مؤقت يكفي للاختبار؛ الترخيص الكامل مطلوب للإنتاج.  
- **ما إصدارات .NET المدعومة؟** .NET Framework 4.5+، .NET Core 3.1+، .NET 5/6/7.

## ما هي مجموعة بيانات File GDB؟
File Geodatabase (GDB) هي مخزن بيانات يعتمد على المجلدات يحتوي على طبقات GIS، جداول، وعلاقات. باستخدام Aspose.GIS يمكنك برمجيًا **إنشاء مجموعة بيانات File GDB** دون الحاجة إلى تثبيت ArcGIS، مما يجعلها مثالية للخطوط الأوتوماتيكية أو التطبيقات المخصصة.

## لماذا نعيّن التحملات للطبقة؟
تعيين التحملات يضمن أن حسابات الهندسة (مثل التقاطعات، التوسيع، أو الالتقاط) تحترم الدقة التي تحتاجها. هذا مهم بشكل خاص عند التعامل مع بيانات عالية الدقة أو عند التصدير إلى منصات GIS أخرى تتوقع قيم تحمل محددة.

## المتطلبات المسبقة
قبل أن نغوص في الشيفرة، تأكد من توفر ما يلي:

- **Aspose.GIS for .NET Library** – قم بتحميل وتثبيت مكتبة Aspose.GIS من [رابط التحميل](https://releases.aspose.com/gis/net/). إذا لم تقم بالحصول عليها بعد، يمكنك استكشاف المكتبة أكثر في [الوثائق](https://reference.aspose.com/gis/net/).
- **بيئة التطوير** – Visual Studio، Rider، أو أي IDE يدعم تطوير .NET.
- **ترخيص صالح** – استخدم ترخيصًا مؤقتًا للاختبار أو ترخيصًا كاملاً للإنتاج (انظر الروابط في قسم الأسئلة الشائعة).

الآن بعد أن أصبح كل شيء جاهزًا، لنستورد المساحات الاسمية التي سنحتاجها.

## استيراد المساحات الاسمية
في تطبيق .NET الخاص بك، أضف المساحات الاسمية التالية للاستفادة من وظائف Aspose.GIS:

```csharp
using Aspose.Gis;
using Aspose.Gis.Formats.FileGdb;
using Aspose.Gis.Geometries;
using Aspose.Gis.SpatialReferencing;
using System;
using System.Text;
```

مع وجود المساحات الاسمية، يمكننا البدء في بناء مجموعة البيانات.

## دليل خطوة بخطوة

### الخطوة 1: تحديد دليل المستند الخاص بك
أولًا، وجه الشيفرة إلى المجلد الذي تريد إنشاء File GDB فيه:

```csharp
string dataDir = "Your Document Directory";
```

> **نصيحة احترافية:** استخدم `Path.Combine` إذا كنت بحاجة إلى بناء المسار بطريقة مستقلة عن النظام.

### الخطوة 2: إنشاء مجموعة بيانات File GDB
الآن نقوم فعليًا **بإنشاء مجموعة بيانات File GDB** على القرص. طريقة `Dataset.Create` تأخذ المسار الكامل ونوع السائق (`Drivers.FileGdb`).

```csharp
var path = dataDir + "TolerancesForFileGdbLayer_out.gdb";
using (var dataset = Dataset.Create(path, Drivers.FileGdb))
{
```

> يضمن كتلة `using` إغلاق مجموعة البيانات بشكل صحيح وتفريغها إلى القرص عند الانتهاء.

### الخطوة 3: تعيين التحملات باستخدام `FileGdbOptions`
قبل إنشاء طبقة، عرّف التحملات التي تحتاجها. يتيح لك `FileGdbOptions` تحديد تحملات XY و Z و M.

```csharp
var options = new FileGdbOptions
{
    XYTolerance = 0.001,
    ZTolerance = 0.1,
    MTolerance = 0.1,
};
```

هذه القيم نموذجية للبيانات الهندسية عالية الدقة، لكن يمكنك تعديلها لتناسب مشروعك.

### الخطوة 4: إنشاء طبقة بالتحملات المحددة
أخيرًا، أنشئ طبقة جديدة داخل مجموعة البيانات، مع تمرير كائن الخيارات الذي قمنا بتكوينه للتو.

```csharp
using (var layer = dataset.CreateLayer("layer_name", options))
{
    // The layer is created with the provided tolerances, ready for use in ArcGIS features/tools.
}
```

عند انتهاء كتلة `using`، تُحفظ الطبقة بالتحملات التي حددتها.
