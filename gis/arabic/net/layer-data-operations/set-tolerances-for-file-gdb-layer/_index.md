# كيفية إنشاء مجموعة بيانات GDB وتعيين التحملات لطبقة

## مقدمة
إذا كنت بحاجة إلى **إنشاء مجموعة بيانات ملف GDB** والتحكم في دقتها، فأنت في المكان الصحيح. في هذا الدرس سنستعرض العملية بالكامل—بدءًا من إعداد مشروع .NET الخاص بك، وإنشاء مجموعة بيانات File Geodatabase (GDB)، ثم تطبيق التحملات XY و Z و M على طبقة جديدة. في النهاية ستحصل على مجموعة بيانات جاهزة للاستخدام تعمل بسلاسة مع أدوات ArcGIS وتطبيقات GIS الأخرى. يوضح هذا الدليل **كيفية إنشاء ملفات gdb** برمجيًا، بحيث يمكنك أتمتة خطوط البيانات دون تدخل يدوي.

## إجابات سريعة
- **ماذا يعني “create file GDB dataset”؟** ينشئ حاوية File Geodatabase جديدة على القرص يمكنها احتواء طبقات GIS متعددة.  
- **لماذا تعيين التحملات؟** التعامُل مع التحملات يحدد الدقة لعمليات الهندسة، مما يمنع أخطاء التقريب في التحليل المكاني.  
- **أي فئة من Aspose.GIS تُستخدم؟** `Dataset.Create` مع `FileGdbOptions`.  
- **هل أحتاج إلى ترخيص للتطوير؟** الترخيص المؤقت يكفي للاختبار؛ الترخيص الكامل مطلوب للإنتاج.  
- **ما إصدارات .NET المدعومة؟** .NET Framework 4.5+, .NET Core 3.1+, .NET 5/6/7.

## ما هي مجموعة بيانات File GDB؟
File Geodatabase (GDB) هو مخزن بيانات يعتمد على المجلدات يحتوي على طبقات GIS وجداول وعلاقات. باستخدام Aspose.GIS يمكنك برمجيًا **إنشاء مجموعة بيانات ملف GDB** دون الحاجة إلى تثبيت ArcGIS، مما يجعلها مثالية للخطوط الأوتوماتيكية أو التطبيقات المخصصة.

## لماذا تعيين التحملات لطبقة؟
تعيين التحملات يضمن أن حسابات الهندسة (مثل التقاطعات، التوسيع، أو الالتقاط) تحترم الدقة التي تحتاجها. هذا مهم بشكل خاص عند التعامل مع بيانات عالية الدقة أو عند تصديرها إلى منصات GIS أخرى تتوقع قيم تحمل محددة. بعبارة أخرى، أنت **تحدد دقة الطبقة** لتجنب أخطاء الهندسة غير المتوقعة.

## استيراد مساحات الأسماء
في تطبيق .NET الخاص بك، قم بتضمين مساحات الأسماء التالية للاستفادة من وظائف Aspose.GIS:

```csharp
using Aspose.Gis;
using Aspose.Gis.Formats.FileGdb;
using Aspose.Gis.Geometries;
using Aspose.Gis.SpatialReferencing;
using System;
using System.Text;
```

مع وجود مساحات الأسماء، يمكننا البدء في بناء مجموعة البيانات.

## كيفية إنشاء مجموعة بيانات GDB؟
فيما يلي دليل خطوة بخطوة يوضح لك كيفية إنشاء مجموعة البيانات وتكوين التحملات.

### الخطوة 1: تحديد دليل المستند الخاص بك
أولاً، وجه الشيفرة إلى المجلد الذي تريد إنشاء File GDB فيه:

```csharp
string dataDir = "Your Document Directory";
```

> **نصيحة احترافية:** استخدم `Path.Combine` إذا كنت بحاجة إلى بناء المسار بطريقة مستقلة عن النظام.

### الخطوة 2: إنشاء مجموعة بيانات File GDB
الآن نقوم فعليًا **بإنشاء مجموعة بيانات ملف GDB** على القرص. طريقة `Dataset.Create` تأخذ المسار الكامل ونوع السائق (`Drivers.FileGdb`).

```csharp
var path = dataDir + "TolerancesForFileGdbLayer_out.gdb";
using (var dataset = Dataset.Create(path, Drivers.FileGdb))
{
```

> يضمن كتلة `using` إغلاق مجموعة البيانات بشكل صحيح وتفريغها إلى القرص عند الانتهاء.

### الخطوة 3: تعيين التحملات باستخدام `FileGdbOptions`
قبل إنشاء طبقة، حدد التحملات التي تحتاجها. يتيح لك `FileGdbOptions` تحديد تحملات XY و Z و M—هذا هو كائن **file gdb options** الذي يتحكم في الدقة.

```csharp
var options = new FileGdbOptions
{
    XYTolerance = 0.001,
    ZTolerance = 0.1,
    MTolerance = 0.1,
};
```

هذه القيم نموذجية لبيانات الهندسة عالية الدقة، ولكن يمكنك تعديلها لتناسب مشروعك.

### الخطوة 4: إنشاء طبقة GIS مع التحملات المحددة
أخيرًا، أنشئ طبقة جديدة داخل مجموعة البيانات، مع تمرير كائن الخيارات الذي قمنا بتكوينه للتو. تُظهر هذه الخطوة **كيفية تعيين التحملات** بالإضافة إلى **إنشاء طبقة GIS**.

```csharp
using (var layer = dataset.CreateLayer("layer_name", options))
{
    // The layer is created with the provided tolerances, ready for use in ArcGIS features/tools.
}
```

عند انتهاء كتلة `using`، يتم حفظ الطبقة مع التحملات التي حددتها.
