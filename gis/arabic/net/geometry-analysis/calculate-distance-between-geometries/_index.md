# كيفية حساب المسافة بين الأشكال الهندسية باستخدام Aspose.GIS

## مقدمة
إذا احتجت يومًا إلى معرفة **كيفية حساب المسافة** بين كائنين مكانيين — سواء كان ذلك شبكة طرق، أو مناطق توصيل، أو ميزات بيئية — فهذا الدليل لك. في .NET، تجعل Aspose.GIS حساب المسافات بسيطًا، دقيقًا، وعالي الأداء. سنستعرض مثالًا واقعيًا يوضح **كيفية استخدام Aspose.GIS**، إنشاء أشكال هندسية بسيطة، واستدعاء طريقة **GetDistanceTo** للحصول على المسافة بينهما.

## إجابات سريعة
- **ماذا يعني “حساب المسافة” في نظم المعلومات الجغرافية؟** يُعيد أقصر مسافة (إقليدية) بين شكلين هندسيين.  
- **أي فئة في Aspose.GIS توفر عملية الحساب؟** `Geometry.GetDistanceTo(Geometry other)`.  
- **هل أحتاج إلى ترخيص للتطوير؟** النسخة التجريبية المجانية تكفي للاختبار؛ الترخيص مطلوب للإنتاج.  
- **هل يمكنني حساب المسافة للأشكال الهندسية ثلاثية الأبعاد؟** نعم، تدعم Aspose.GIS الحسابات ثنائية وثلاثية الأبعاد.  
- **ما إصدارات .NET المدعومة؟** .NET Framework 4.6+، .NET Core 3.1+، .NET 5/6/7.

## ما هو حساب المسافة في برمجة الجغرافيا المكانية؟
يقيس حساب المسافة أقصر خط يربط بين شكلين هندسيين. وهو عملية أساسية للمهام مثل تحليل القرب، التوجيه، التجميع، والفهرسة المكانية.

## لماذا نستخدم Aspose.GIS لحساب المسافة؟
- **دقة عالية** – يستخدم حسابًا بدقة مزدوجة.  
- **بدون تبعيات** – لا يتطلب مكتبات GIS أصلية.  
- **متعدد المنصات** – يعمل على Windows وLinux وmacOS مع .NET Core/5+.  
- **نموذج هندسي غني** – يدعم النقاط، الخطوط، المضلعات، والأشكال المتعددة مباشرةً.

## المتطلبات المسبقة
- **Aspose.GIS for .NET** مثبت (حمّل من [صفحة إصدارات Aspose.GIS for .NET](https://releases.aspose.com/gis/net/)).  
- معرفة أساسية بـ C# وبنية مشروع .NET.  
- بيئة تطوير مثل Visual Studio 2022 أو VS Code.

## استيراد مساحات الأسماء
قبل أن تبدأ باستخدام Aspose.GIS، أضف مساحات الأسماء المطلوبة إلى ملف C# الخاص بك:

```csharp
using Aspose.Gis.Geometries;
using System;
using System.Collections.Generic;
using System.Linq;
using System.Text;
using System.Threading.Tasks;
```

### الخطوة 1: إنشاء شكل هندسي من نوع مضلع
```csharp
var polygon = new Polygon();
```
نبدأ بمضلع فارغ سيُمثل لاحقًا مساحة مستطيلة.

### الخطوة 2: تعريف الحلقة الخارجية للمضلع
```csharp
polygon.ExteriorRing = new LinearRing(new[]
{
    new Point(0, 0),
    new Point(0, 1),
    new Point(1, 1),
    new Point(1, 0),
    new Point(0, 0),
});
```
الحلقة الخارجية هي حلقة مغلقة من النقاط تُحدد حدود المضلع. في هذا المثال ننشئ مربعًا بحجم 1 × 1.

### الخطوة 3: إنشاء شكل هندسي من نوع LineString
```csharp
var line = new LineString();
```
`LineString` هو سلسلة من القطع الخطية المتصلة. سنستخدمه لتمثيل طريق أو مسار.

### الخطوة 4: إضافة نقاط إلى الـ LineString
```csharp
line.AddPoint(2, 0);
line.AddPoint(1, 3);
```
هاتان النقطتان تمنحان الخط شكلًا مائلًا لا يتقاطع مع المضلع، مما يجعل حساب المسافة مثيرًا للاهتمام.

### الخطوة 5: حساب المسافة
```csharp
double distance = polygon.GetDistanceTo(line);
```
هنا نقوم **بالحصول على المسافة إلى الشكل الهندسي** عبر استدعاء `GetDistanceTo`. تقوم Aspose.GIS بحساب أقصر مسافة بين حافة المضلع والخط.

### الخطوة 6: إخراج النتيجة
```csharp
Console.WriteLine(distance.ToString("F")); // 0.63
```
يتم طباعة النتيجة بدقتين عشريتين (`0.63`). تمثل هذه القيمة الحد الأدنى للمسافة الإقليدية بين المربع والخط.

## حالات الاستخدام الشائعة
- **اللوجستيات:** العثور على أقرب مستودع إلى مسار التوصيل.  
- **المراقبة البيئية:** قياس مسافة سحابة الملوثات عن منطقة محمية.  
- **التخطيط الحضري:** تحديد المسافة بين البنية التحتية المقترحة والمعالم القائمة.

## استكشاف الأخطاء وإصلاحها والنصائح
- **نظام الإحداثيات مهم:** تأكد من أن كلا الشكلين يستخدمان نفس نظام الإحداثيات المرجعي (CRS) قبل حساب المسافة.  
- **الأداء:** بالنسبة لمجموعات البيانات الكبيرة، فكر في الفهرسة المكانية (R‑tree) لتجنب المقارنات من الدرجة O(N²).  
- **الدقة:** إذا كنت بحاجة إلى مسافات جيوديسية (دائرية كبرى)، حوّل الإحداثيات إلى إسقاط مناسب أولاً.

## الأسئلة المتداولة

**س: ما مدى دقة المسافة التي تُعيدها `GetDistanceTo`؟**  
ج: تستخدم الطريقة حسابًا بدقة مزدوجة وتلتزم بمواصفات OGC Simple Features، مما يوفر دقة أقل من المتر للإحداثيات المستوية النموذجية.

**س: هل يمكنني حساب المسافة بين `Point` و `Polygon`؟**  
ج: نعم — فقط استدعِ `point.GetDistanceTo(polygon)` (أو العكس) وستُعيد Aspose.GIS أقصر مسافة من النقطة إلى حافة المضلع.

**س: هل يدعم الـ API حسابات المسافة على دفعات؟**  
ج: رغم عدم وجود طريقة دفعة واحدة، يمكنك التكرار عبر مجموعات من الأشكال الهندسية وإعادة استخدام استدعاء `GetDistanceTo` بشكل فعال.

**س: ماذا يحدث إذا تقاطعت الأشكال الهندسية؟**  
ج: تُعيد الطريقة `0.0` لأن أقصر مسافة بين الأشكال المتقاطعة هي صفر.

**س: هل هناك طريقة للحصول على أقرب نقاط على كل شكل هندسي؟**  
ج: نعم — استخدم `Geometry.GetNearestPoints(Geometry other)` التي تُعيد زوجًا يحتوي على أقرب نقطتين على كلا الشكلين.
