# كيفية تحويل الهندسة إلى WKT باستخدام Aspose.GIS لـ .NET

## مقدمة
إذا كنت تعمل مع البيانات المكانية في تطبيق .NET، فستحتاج غالبًا إلى **ترجمة الهندسة** إلى تمثيل نصي يمكن للأنظمة الأخرى استهلاكه. تنسيق النص المعروف (WKT) هو المعيار الفعلي لهذا الغرض. في هذا الدرس سنستعرض **كيفية ترجمة الهندسة** إلى WKT باستخدام Aspose.GIS لـ .NET، وسنُظهر لك أيضًا طريقة `AsText()` المفيدة التي تجعل التحويل سطرًا واحدًا.

## إجابات سريعة
- **ماذا يعني “ترجمة الهندسة”؟** تحويل كائن الهندسة (نقطة، خط، مضلع، إلخ) إلى تنسيق نصي مثل WKT.  
- **أي طريقة تُنشئ WKT؟** `AsText()` على أي كائن هندسة.  
- **هل أحتاج إلى ترخيص؟** النسخة التجريبية المجانية تعمل للتطوير؛ يتطلب الترخيص التجاري للإنتاج.  
- **الإصدارات المدعومة من .NET؟** .NET Framework 4.5+، .NET Core 3.1+، .NET 5/6+.  
- **هل يمكنني تحويل صيغ أخرى؟** نعم – يدعم Aspose.GIS أيضًا WKB، GeoJSON، Shapefile، وأكثر.

## ما هي ترجمة الهندسة إلى WKT؟
تعني ترجمة الهندسة إلى WKT تمثيل إحداثيات وشكل الكائن المكاني كسلسلة نصية عادية، مثل `POINT (23.5732 25.3421)`. هذا التنسيق قابل للقراءة البشرية ومقبول على نطاق واسع من قبل أدوات GIS، قواعد البيانات، وخدمات الويب.

## لماذا تستخدم Aspose.GIS لهذه المهمة؟
* **واجهة برمجة تطبيقات بدون تبعيات** – لا توجد مكتبات أصلية للتثبيت.  
* **سلوك ثابت** عبر .NET Framework، .NET Core، و .NET 5/6.  
* **دعم صيغ غني** – بالإضافة إلى WKT تحصل على WKB، GeoJSON، Shapefile، إلخ.  
* **آمن من حيث الخيوط وعالي الأداء** – مثالي لكل من السكريبتات الصغيرة والخدمات واسعة النطاق.

## كيفية ترجمة الهندسة إلى WKT باستخدام Aspose.GIS لـ .NET
في الأقسام أدناه نقسم العملية إلى خطوات واضحة مرقمة. كل خطوة تتضمن شرحًا قصيرًا يليه الشيفرة الدقيقة التي تحتاجها.

### الخطوة 1: استيراد المساحات الاسمية المطلوبة
أولاً، استدعِ فئات الهندسة الخاصة بـ Aspose.GIS إلى النطاق.

```csharp
using Aspose.Gis.Geometries;
using System;
using System.Collections.Generic;
using System.Linq;
using System.Text;
using System.Threading.Tasks;
```

### الخطوة 2: إنشاء كائن هندسة (مثال نقطة)
أنشئ الهندسة التي تريد ترجمتها. هنا نستخدم `Point`، لكن النمط نفسه يعمل مع `LineString`، `Polygon`، إلخ.

```csharp
Point point = new Point(23.5732, 25.3421);
```

### الخطوة 3: تحويل الهندسة إلى WKT باستخدام `AsText()`
طريقة الامتداد `AsText()` تُعيد تمثيل WKT للهندسة. اطبعها إلى وحدة التحكم أو احفظها حسب الحاجة.

```csharp
Console.WriteLine(point.AsText()); // POINT (23.5732, 25.3421)
```

> **نصيحة احترافية:** إذا كنت بحاجة إلى WKT بدون أقواس حول الإحداثيات، استخدم `point.AsText().Replace(",", " ")`.

## كيفية استخدام طريقة AsText
`AsText()` هي الطريقة الأساسية **لتحويل الهندسة إلى WKT**. تعمل على أي فئة مشتقة من `Geometry`، لذا يمكنك استدعاؤها مباشرة على `LineString`، `Polygon`، `MultiPolygon`، إلخ، دون خطوات تحويل إضافية.

## المشكلات الشائعة والحلول
| المشكلة | السبب | الحل |
|-------|--------|-----|
| `AsText()` يُعيد `null` | لم يتم تهيئة الهندسة | تأكد من إنشاء كائن الهندسة بإحداثيات صالحة قبل استدعاء `AsText()`. |
| تنسيق غير متوقع (فاصلة مقابل مسافة) | أدوات GIS المختلفة تتوقع فواصل مختلفة | استخدم تعديل السلسلة (`Replace`) أو فئة `WktWriter` للتنسيق المخصص. |
| عنق زجاجة في الأداء عند تحويل مجموعات كبيرة | إدخال/إخراج المتحكم المتكرر | قم بالتحويل على دفعات واكتب إلى ملف أو `StringBuilder` بدلاً من `Console.WriteLine`. |

## الخلاصة
ترجمة الهندسة إلى WKT باستخدام Aspose.GIS لـ .NET أمر بسيط: استورد المساحات الاسمية، أنشئ الهندسة الخاصة بك، واستدعِ `AsText()`. يتيح لك هذا النهج دمج قدرات GIS مباشرةً في تطبيقات .NET الخاصة بك دون تبعيات خارجية.

## أسئلة شائعة
**س: كيف يمكنني تحويل مجموعة من الهندسات إلى WKT بشكل فعال؟**  
ج: قم بالتكرار عبر المجموعة واستدعِ `AsText()` على كل عنصر، واحفظ النتائج في `StringBuilder` أو اكتبها مباشرةً إلى ملف لتجنب عبء وحدة التحكم.  

**س: ماذا لو احتجت لتصدير WKT بمعرف مرجعي مكاني (SRID) محدد؟**  
ج: استخدم النسخة المتعددة `AsText(Srid)` حيث تزود المعرف المرجعي المكاني المطلوب.  

**س: هل طريقة `AsText()` حساسة للغة المحلية؟**  
ج: `AsText()` دائمًا تستخدم الثقافة غير المتغيرة، مما يضمن فواصل عشرية ثابتة بغض النظر عن لغة النظام.  

**س: هل يمكنني تحليل WKT مرة أخرى إلى كائن هندسة؟**  
ج: نعم، استخدم `Geometry.FromText(string wkt)` لإنشاء كائن هندسة من سلسلة WKT.  

**س: هل يدعم Aspose.GIS إحداثيات ثلاثية الأبعاد في WKT؟**  
ج: بدءًا من الإصدار 22.10، تدعم المكتبة قيم Z و M في WKT (مثال: `POINT Z (x y z)`).
