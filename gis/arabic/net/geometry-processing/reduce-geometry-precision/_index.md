# كيفية تقليل دقة الهندسة وتدوير Z في .NET

## مقدمة
إذا كنت تعمل مع مجموعات بيانات مكانية كبيرة، فمن المحتمل أنك لاحظت أن كل خانة عشرية إضافية في بيانات الهندسة تتراكم – سواء في حجم الملف أو في وقت المعالجة. في هذا البرنامج التعليمي ستتعلم **كيفية تقليل دقة الهندسة** و**كيفية تدوير قيم Z** باستخدام Aspose.GIS لـ .NET. في نهاية الدليل ستكون قادرًا على تقليل حجم ملفات الهندسة، وتسريع العمليات المكانية، والحفاظ على استهلاك الذاكرة منخفضًا، كل ذلك عبر عدد قليل من استدعاءات الطرق البسيطة.

## إجابات سريعة
- **ماذا يعني “كيفية تدوير Z”؟** إنه يزيل عدد الخانات العشرية لإحداثية Z في كائن الهندسة.  
- **لماذا تقليل دقة الهندسة؟** إنه يقلل كمية البيانات المخزنة لكل رأس، مما يسرّع الاستعلامات المكانية ويقلل من استهلاك الذاكرة.  
- **أي مكتبة تتعامل مع هذا؟** Aspose.GIS لـ .NET توفر طرقًا مدمجة `RoundZ` و `RoundXY`.  
- **هل أحتاج إلى ترخيص؟** النسخة التجريبية المجانية تعمل للاختبار؛ الترخيص التجاري مطلوب للإنتاج.  
- **هل يمكنني التحكم في عدد الخانات العشرية؟** نعم، يمكنك تحديد عدد الأرقام المطلوبة في طرق `Round*`.

## كيفية تقليل دقة الهندسة في .NET
تقليل دقة الهندسة بسيط كما هو الحال عند استدعاء `RoundXY` على أي كائن هندسي. تقبل الطريقة عدد الخانات العشرية التي تريد الاحتفاظ بها لإحداثيات X و Y. هذه العملية مفيدة بشكل خاص عندما لا تكون الدقة الفرعية بالمتر ضرورية لتحليلك.

## كيفية تدوير قيم Z في .NET
عندما تتضمن بياناتك مكوّن Z (الارتفاع)، يمكنك استدعاء `RoundZ` لتحديد دقته. هذه هي خطوة **كيفية تدوير Z** التي غالبًا ما تحقق أكبر تخفيض في حجم الملفات لمجموعات البيانات ثلاثية الأبعاد، لأن قيم الارتفاع تميل إلى امتلاك العديد من الخانات العشرية.

## ما هو “كيفية تدوير Z” في نظم المعلومات الجغرافية؟
تدوير إحداثية Z يزيل الدقة العشرية الزائدة، محولًا قيمة مثل 3.345 إلى 3.3 (أو أي دقة تحددها). يمكن لهذه العملية البسيطة أن تقلص حجم الملفات بشكل كبير وتسرّع الحسابات، خاصة عندما لا تكون البُعد الثالث حاسمًا لتحليلك.

## لماذا تقليل دقة الهندسة باستخدام Aspose.GIS؟
- **زيادة الأداء:** بيانات أقل للمعالجة تعني استعلامات مكانية وتحويلات أسرع.  
- **توفير الذاكرة:** تمثيلات رؤوس أصغر تحرّر الذاكرة، مما يسمح لمجموعات بيانات أكبر بالبقاء في الذاكرة.  
- **المرونة:** أنت تقرّر مستوى الدقة الذي يوازن بين الدقة والسرعة.

## استيراد مساحات الأسماء
أولاً، استورد مساحات الأسماء اللازمة لاستخدام فئات وأساليب Aspose.GIS.

```csharp
using Aspose.Gis.Geometries;
using System;
using System.Collections.Generic;
using System.Linq;
using System.Text;
using System.Threading.Tasks;
```

## الخطوة 1: إنشاء نقطة
لنبدأ بإنشاء نقطة بإحداثيات محددة.

```csharp
Point point = new Point(1.344, 2.345, 3.345, 4.345);
```

## الخطوة 2: تقليل دقة XY
الآن، سنقلل دقة إحداثيات X و Y للنقطة إلى خانتين عشريتين.

```csharp
point.RoundXY(digits: 2);
```

## الخطوة 3: عرض الإحداثيات
عرض الإحداثيات المحدثة للنقطة.

```csharp
Console.WriteLine("{0}, {1}, {2}, {3}", point.X, point.Y, point.Z, point.M);
```

## الخطوة 4: تقليل دقة Z – **كيفية تدوير Z**
بعد ذلك، لنقلل دقة إحداثية Z للنقطة إلى خانة عشرية واحدة. هذه هي جوهر **كيفية تدوير Z**.

```csharp
point.RoundZ(digits: 1);
```

## الخطوة 5: عرض الإحداثيات المحدثة
عرض الإحداثيات المحدثة للنقطة بعد تقليل دقة Z.

```csharp
Console.WriteLine("{0}, {1}, {2}, {3}", point.X, point.Y, point.Z, point.M);
```

## الخطوة 6: إنشاء LineString
الآن، لننشئ `LineString` ونضيف نقاطًا إليه.

```csharp
LineString line = new LineString();
line.AddPoint(1.2, 2.3);
line.AddPoint(2.4, 3.1);
```

## الخطوة 7: تقليل دقة XY لـ LineString
قلل دقة إحداثيات X و Y لـ `LineString` إلى صفر خانات عشرية.

```csharp
line.RoundXY(digits: 0);
```

## الخطوة 8: عرض الإحداثيات المحدثة لـ LineString
عرض الإحداثيات المحدثة لـ `LineString` بعد تقليل دقة XY.

```csharp
Console.WriteLine("{0}, {1}", line[0].X, line[0].Y);
Console.WriteLine("{0}, {1}", line[1].X, line[1].Y);
```

## حالات الاستخدام الشائعة والنصائح
- **تحويلات رستر‑فكتور الكبيرة:** تدوير Z يمكن أن يقلص ملفات الهندسة الوسيطة.  
- **تطبيقات GIS المحمولة:** دقة أقل تقلل من عرض النطاق الترددي عند نقل الهندسة عبر الشبكة.  
- **نصيحة احترافية:** تطبيق `RoundXY` قبل `RoundZ` للحفاظ على تناسق سير العمل.
