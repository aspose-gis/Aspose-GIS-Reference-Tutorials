# Преобразование WKT в Geometry с помощью Aspose.GIS в .NET

## Введение
В этом руководстве вы узнаете **как преобразовать WKT в geometry** с помощью Aspose.GIS для .NET и увидите практический пример **как подсчитать точки** в полученной фигуре. Независимо от того, создаёте ли вы сервис картографии, обрабатываете GIS‑данные или просто нужно прочитать Well‑Known Text в .NET‑приложении, эти шаги быстро помогут вам начать работу.

## Быстрые ответы
- **Может ли Aspose.GIS читать WKT?** Да — метод `Geometry.FromText` напрямую разбирает строки WKT.  
- **Сколько строк кода требуется?** Около 5‑6 строк для базового преобразования и подсчёта точек.  
- **Нужна ли лицензия для продакшн?** Да, коммерческая лицензия требуется для использования не в режиме пробной версии.  
- **Поддерживаемые версии .NET?** .NET Framework 4.6+, .NET Core 3.1+, .NET 5/6+.  
- **Есть ли встроенный подсчёт точек?** Конечно — объекты geometry предоставляют свойство `Count`.

## Что означает «преобразовать WKT в geometry»?
Well‑Known Text (WKT) — это текстовая разметка для представления геометрических объектов. Преобразование WKT в geometry означает превращение этого текста в объектную модель (например, `ILineString`, `IPolygon`), которой можно управлять программно.

## Почему использовать Aspose.GIS для этого преобразования?
- **Парсинг без зависимостей** — не требуется внешних библиотек.  
- **Полный набор функций GIS** — поддерживает 2D/3D координаты, работу с SRID и множество форматов файлов.  
- **Высокая производительность** — оптимизировано для больших наборов данных и многопоточных сценариев.

## Импорт пространств имён
Сначала добавьте необходимые пространства имён в ваш файл C#:

```csharp
using Aspose.Gis.Geometries;
using System;
using System.Collections.Generic;
using System.Linq;
using System.Text;
using System.Threading.Tasks;
```

## Шаг 1: Создать LineString из WKT
Теперь мы **преобразуем WKT в geometry**, создав объект `LineString` из строки WKT, содержащей координаты Z:

```csharp
ILineString line = (ILineString)Geometry.FromText("LINESTRING Z (0.1 0.2 0.3, 1 2 1, 12 23 2)");
```

> *Pro tip:* Метод `FromText` автоматически определяет тип геометрии, поэтому вы можете привести его к соответствующему интерфейсу (`ILineString`, `IPolygon` и т.д.).

## Как подсчитать точки в LineString?
Чтобы ответить на вторичный запрос **как подсчитать точки**, просто прочитайте свойство `Count` у экземпляра `ILineString`:

```csharp
Console.WriteLine(line.Count); // Output: 3
```

Вывод показывает, что линия содержит три вершины, подтверждая, что преобразование прошло успешно и количество точек корректно.

## Часто задаваемые вопросы (дополнительно)

**В: Что делать, если моя строка WKT содержит неверный синтаксис?**  
**О:** `Geometry.FromText` бросает `FormatException`. Оберните вызов в блок try‑catch, чтобы корректно обрабатывать некорректный WKT.

**В: Могу ли я преобразовать коллекцию строк WKT за один проход?**  
**О:** Да — пройдитесь по списку строк, вызовите `Geometry.FromText` для каждого элемента и сохраните результаты в `List<IGeometry>`.

**В: Сохраняет ли Aspose.GIS значения Z при преобразовании?**  
**О:** Абсолютно. Когда WKT включает координату Z (как показано в примере), полученная geometry сохраняет эти значения.

**В: Можно ли экспортировать geometry обратно в WKT после манипуляций?**  
**О:** Используйте метод `ToText()` у любого экземпляра `IGeometry`, чтобы получить представление в формате WKT.
