# Как создать коллекцию геометрий и перебрать геометрии в .NET

## Введение
В области обработки и анализа геопространственных данных Aspose.GIS для .NET выступает мощным набором инструментов, позволяя разработчикам **создавать коллекцию геометрий** объектов, **обрабатывать геопространственные данные** и визуализировать географическую информацию без проблем в приложениях .NET. Эта статья служит всесторонним руководством по эффективному использованию Aspose.GIS для .NET, предназначенным как для начинающих, так и для опытных разработчиков.

## Быстрые ответы
- **Что я могу достичь?** Создать коллекцию геометрий, добавить точечную геометрию и перебрать каждый тип геометрии.  
- **Какая библиотека требуется?** Aspose.GIS для .NET (последний релиз).  
- **Нужна ли лицензия?** Доступна временная лицензия для оценки; полная лицензия требуется для продакшн.  
- **Какие версии .NET поддерживаются?** Работает с .NET Framework, .NET Core и .NET 5/6+.  
- **Сколько времени занимает реализация?** Обычно менее 15 минут для базового рабочего процесса с коллекцией.

## Что такое коллекция геометрий?
**Коллекция геометрий** — это контейнер, который может хранить несколько геометрических объектов — точки, линии, полигоны и т.д., позволяя рассматривать их как единое целое. Это особенно полезно, когда необходимо **обрабатывать геопространственные данные** в приложениях .NET, которые включают смешанные типы геометрий.

## Зачем создавать коллекцию геометрий?
- **Упрощает итерацию** — вы можете проходить по разнородным геометриям с помощью единственного `foreach`.  
- **Поддерживает порядок данных** — группируйте связанные объекты без создания отдельных контейнеров.  
- **Позволяет выполнять массовые операции** — применять трансформации или анализ ко всем элементам за один проход.

## Импорт пространств имён
Начните с импорта необходимых пространств имён, чтобы эффективно получить доступ к функциям, предоставляемым Aspose.GIS для .NET.

```csharp
using Aspose.Gis.Geometries;
using System;
using System.Collections.Generic;
using System.Linq;
using System.Text;
using System.Threading.Tasks;
```

Теперь разберём предоставленный пример на несколько шагов, чтобы понять процесс **создания коллекции геометрий** и перебора её геометрий с помощью Aspose.GIS для .NET.

## Шаг 1: Создание геометрических объектов
Создайте точечные и линейные геометрии, используя предоставленные координаты. Это демонстрирует, как **добавить точечную геометрию** в коллекцию.

```csharp
Point pointGeometry = new Point(40.7128, -74.006);
LineString lineGeometry = new LineString();
lineGeometry.AddPoint(78.65, -32.65);
lineGeometry.AddPoint(-98.65, 12.65);
```

## Шаг 2: Заполнение коллекции геометрий
Создайте коллекцию геометрий и добавьте в неё созданные геометрии. Это ядро **создания коллекции геометрий**.

```csharp
GeometryCollection geometryCollection = new GeometryCollection();
geometryCollection.Add(pointGeometry);
geometryCollection.Add(lineGeometry);
```

## Шаг 3: Перебор геометрий
Пройдитесь по коллекции геометрий и обработайте каждую геометрию в зависимости от её типа. Этот шаблон идеален для **обработки геопространственных данных**, когда присутствуют смешанные типы геометрий.

```csharp
foreach (Geometry geometry in geometryCollection)
{
    switch (geometry.GeometryType)
    {
        case GeometryType.Point:
            Point point = (Point)geometry;
            // Handle point geometry
            break;
        case GeometryType.LineString:
            LineString line = (LineString)geometry;
            // Handle line geometry
            break;
    }
}
```

## Распространённые ошибки и советы
- **Безопасность приведения типов**: Всегда проверяйте `GeometryType` перед приведением, чтобы избежать исключений во время выполнения.  
- **Порядок координат**: Aspose.GIS ожидает сначала широту, затем долготу; их перемешивание может привести к инвертированным позициям.  
- **Производительность**: Для больших коллекций рассмотрите использование `Parallel.ForEach` для ускорения обработки, но будьте внимательны к потокобезопасности при изменении общих ресурсов.
