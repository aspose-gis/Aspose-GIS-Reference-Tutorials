# Добавление точечного объекта и указание имен полей Object ID и Geometry

## Шаг 1: Добавление точечного объекта и указание имен полей Object ID и Geometry
В этом шаге вы узнаете, как настроить имена полей Object ID и Geometry для ваших GIS‑данных. Это важно для эффективного управления данными и возможности **прочитать object id** позже.

### Шаг 1.1: Установка каталога документов
Начните с определения пути к вашему каталогу документов:

```csharp
string dataDir = "Your Document Directory";
```

### Шаг 1.2: Создание GeoDatabase и определение параметров
Создайте GeoDatabase с желаемыми **именами полей** для Object ID и Geometry:

```csharp
var path = dataDir + "NamesOfObjectIdAndGeometryFields_out.gdb";
using (var dataset = Dataset.Create(path, Drivers.FileGdb))
{
    var options = new FileGdbOptions
    {
        ObjectIdFieldName = "OID",         // Specify the Object ID field name
        GeometryFieldName = "POINT",       // Specify the Geometry field name
    };
```

### Шаг 1.3: Создание и добавление слоя
Создайте слой внутри GeoDatabase и добавьте объект, представляющий точечную геометрию:

```csharp
using (var layer = dataset.CreateLayer("layer_name", options, SpatialReferenceSystem.Wgs84))
{
    var feature = layer.ConstructFeature();
    feature.Geometry = new Point(12.32, 34.21);  // Specify the geometry (in this case, a point)
    layer.Add(feature);
}
```

### Шаг 1.4: Открытие и получение данных из слоя
Откройте слой и получите объект, который вы только что добавили. Это демонстрирует, как **прочитать object id** из набора данных:

```csharp
using (var layer = dataset.OpenLayer("layer_name"))
{
    var feature = layer[0];
    Console.WriteLine(feature.GetValue<int>("OID")); // Output: 1
}
```

## Зачем задавать пользовательские имена полей?
Настройка имен полей Object ID и Geometry предоставляет гибкость при интеграции с существующими базами данных или соблюдении соглашений об именовании, требуемых downstream‑приложениями. Это также делает данные более самодокументирующимися, что упрощает обслуживание и совместную работу.

## Распространённые подводные камни и устранение неполадок
- **Отсутствующий каталог** — Убедитесь, что `dataDir` указывает на существующую папку; иначе `Dataset.Create` выбросит исключение.
- **Конфликты имён полей** — Если поле с таким именем уже существует в геобазе, создание завершится неудачей. Выберите уникальные имена.
- **Несоответствие пространственной привязки** — Всегда согласовывайте систему пространственной привязки (например, WGS84) с данными, которые вы собираетесь хранить.

## Дополнительные часто задаваемые вопросы

**В: Можно ли изменить поле Object ID после создания слоя?**  
A: Нет. Имя поля задаётся при создании слоя; для его изменения необходимо пересоздать слой с новым объектом `FileGdbOptions`.

**В: Как получить другие значения атрибутов, помимо Object ID?**  
A: Используйте `feature.GetValue<T>("FieldName")`, где `FieldName` — имя атрибута, который вы хотите прочитать.

**В: Можно ли добавить несколько точечных объектов за один пакет?**  
A: Да. Пройдитесь по вашим данным в цикле, создайте объект для каждой точки, задайте её геометрию и вызовите `layer.Add(feature)` внутри того же блока `using`.

**В: Поддерживает ли Aspose.GIS другие типы геометрий?**  
A: Конечно. Вы можете работать с `LineString`, `Polygon`, `MultiPoint` и другими, создавая соответствующие объекты геометрии.

**В: Что произойдёт, если попытаться прочитать Object ID, которого не существует?**  
A: Обращение к индексу вне диапазона (например, `layer[10]`, когда существует только один объект) вызовет `IndexOutOfRangeException`. Сначала всегда проверяйте `layer.Count`.
