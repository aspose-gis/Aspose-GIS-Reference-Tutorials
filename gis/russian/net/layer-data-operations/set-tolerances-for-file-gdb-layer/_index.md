# Создать набор данных File GDB и установить допуски для слоя

## Введение
Если вам нужно **создать набор данных из файла GDB** и контролировать его точность, вы попали в нужное место. В этом руководстве мы пройдем весь процесс — начиная с настройки вашего проекта .NET, создания набора данных файловой базы геоданных (GDB), а затем применения допусков XY, Z и M к новому слою. К концу вы получите готовый набор данных, который будет плавно работать с инструментами ArcGIS и другими ГИС‑приложениями.

## Быстрые ответы
- **Что означает «создать набор данных в файле GDB»?** Он создает новый контейнер файловой базы геоданных на диске, который может хранить несколько ГИС-слоев.
- **Зачем хранить допуски?** Допуски определяют точность геометрических операций, предотвращая ошибки округления при пространственном анализе.
- **Какой класс Aspose.GIS используется?** `Dataset.Create` вместе с `FileGdbOptions`.
- **Нужна ли лицензия на разработку?** Временная лицензия, достаточная для тестирования; Для продакшн требуется полная лицензия.
- **Какие версии .NET используются?** .NET Framework4.5+, .NETCore3.1+, .NET5/6/7.

## Что такое набор данных файла GDB?
Файловая база геоданных (GDB) — это хранилище данных на основе папок, содержащее ГИС-слои, таблицы и связи. С помощью Aspose.GIS вы можете программно **создать набор данных в файле GDB** без необходимости установки ArcGIS, что делает его исправление для рабочих конвейеров или требует приложений.

## Зачем устанавливать допуски для слоя?
Установка допусков гарантирует, что геометрические вычисления (например, пересечения, буферизация или привязка) соблюдают необходимую точность. Это особенно важно при работе с высоким разрешением или при экспорте в других ГИС-платформах, которые прогнозируют важное значение допусков.

Теперь, когда всё готово, имён имён импортируемых необходимых пространств.

## Импортировать пространства имен
В вашем .NET‑приложении имеются следующие пространства, чтобы воспользоваться возможностями Aspose.GIS:

```csharp
using Aspose.Gis;
using Aspose.Gis.Formats.FileGdb;
using Aspose.Gis.Geometries;
using Aspose.Gis.SpatialReferencing;
using System;
using System.Text;
```

С подключенными пространствами мы можем приступить к построению набора данных.

## Пошаговое руководство

### Шаг 1. Определите каталог документов
Сначала укажите в коде папку, в которой будет создан File GDB:

```csharp
string dataDir = "Your Document Directory";
```

> **Совет:** Используйте `Path.Combine`, если нужно построить путь независимо от платформы.

### Шаг 2: Создайте набор данных FileGdb
Теперь мы действительно **создаём набор данных file GDB** на диске. Метод `Dataset.Create` принимает полный путь и тип драйвера (`Drivers.FileGdb`).

```csharp
var path = dataDir + "TolerancesForFileGdbLayer_out.gdb";
using (var dataset = Dataset.Create(path, Drivers.FileGdb))
{
```

> Блок `using` гарантирует, что набор данных будет корректно закрыт и сброшен на диск после завершения работы.

### Шаг 3: Установите допуски с помощью `FileGdbOptions`
Перед созданием слоя задайте необходимые допуски. `FileGdbOptions` позволяет указать допуски XY, Z и M.

```csharp
var options = new FileGdbOptions
{
    XYTolerance = 0.001,
    ZTolerance = 0.1,
    MTolerance = 0.1,
};
```

Эти значения типичны для инженерных данных высокой точности, но вы можете изменить их под требования вашего проекта.

### Шаг 4: Создайте слой с указанными допусками
Наконец, создайте новый слой внутри набора данных, передав объект опций, который мы только что настроили.

```csharp
using (var layer = dataset.CreateLayer("layer_name", options))
{
    // The layer is created with the provided tolerances, ready for use in ArcGIS features/tools.
}
```

Когда блок `using` завершается, слой сохраняется с заданными вами допусками.

## Распространенные проблемы и решения
| Выпуск | Почему это происходит | Исправить |
|-------|----------------|-----|
| **Путь к набору данных не найден** | `dataDir` указывает на несуществующую точку. | Убедитесь, что директория существует, или создайте ее с помощью `Directory.CreateDirectory(dataDir)`. |
| **Недопустимые значения допусков** | Допуски должны быть неотрицательными числами. | Используйте положительные значения; избегайте нуля, если только вы не хотите допустить его отсутствия. |
| **Ошибка лицензии** | Пробная или временная лицензия истекла. | Замените новое временное освещение или полную защиту. |
