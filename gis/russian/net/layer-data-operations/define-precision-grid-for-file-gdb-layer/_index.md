# Как установить сетку для слоя File GDB в Aspose.GIS

## Введение
В этом руководстве вы **создадите объекты файловой геобазы** и научитесь **устанавливать сетку** для слоя File Geodatabase (GDB) с помощью Aspose.GIS для .NET. Определение точной сетки позволяет **проверять диапазон координат**, предотвращает ошибки выхода за пределы и гарантирует, что любая операция **добавления объектов в слой** сохраняет данные точно. Мы пройдём каждый шаг, объясним, почему каждое значение важно, и покажем, как **обрабатывать сценарии выхода за диапазон** корректно.

## Быстрые ответы
- **Что означает «установить сетку»?** Она определяет точность координат и допустимый диапазон для GIS‑слоя.  
- **Зачем использовать точную сетку?** Она защищает ваши данные от недопустимых координат и повышает эффективность хранения.  
- **Какая библиотека предоставляет эту функцию?** Aspose.GIS for .NET.  
- **Нужна ли лицензия?** Доступна пробная версия; для продакшн требуется коммерческая лицензия.  
- **Можно ли использовать это с .NET Core?** Да, Aspose.GIS поддерживает .NET Framework и .NET Core.

## Что такое точная сетка и зачем её устанавливать?
Точная сетка — это набор параметров (начало, масштаб и т.д.), который указывает GIS‑движку, как округлять и хранить значения координат. Настраивая сетку, вы **проверяете диапазон координат** автоматически, и любая попытка вставить точку за пределами сетки вызовет исключение — это помогает **обрабатывать сценарии выхода за диапазон** уже на этапе разработки.

## Зачем создавать файловую геобазу с точной сеткой?
Создание файловой геобазы предоставляет переносимый, высокопроизводительный контейнер для векторных данных. Добавление точной сетки при создании обеспечивает:

- **Последовательное качество данных** — каждый объект соблюдает одинаковую числовую точность.  
- **Более быструю индексацию** — движок может хранить координаты эффективнее.  
- **Раннее обнаружение ошибок** — координаты, выходящие за пределы, отлавливаются до того, как они повредят набор данных.

## Как настроить координатную сетку в слое File GDB
Ниже представлено пошаговое руководство, показывающее, как именно настроить сетку, создать слой и безопасно **добавлять объекты в слой**.

### Шаг 1: Создать набор данных
Мы начинаем с создания нового набора данных File Geodatabase. Здесь будет располагаться слой.

```csharp
var path = "Your Document Directory" + "PrecisionGrid_out.gdb";
using (var dataset = Dataset.Create(path, Drivers.FileGdb))
{
```

### Шаг 2: Определить параметры точной сетки
Здесь мы указываем параметры сетки. Скорректируйте начала и масштабы, чтобы они соответствовали системе координат вашего проекта.

```csharp
var options = new FileGdbOptions
{
    CoordinatePrecisionGrid = new FileGdbCoordinatePrecisionGrid
    {
        XOrigin = -400,
        YOrigin = -400,
        XYScale = 1e10,
        MOrigin = 0,
        MScale = 1e4,
    },
    EnsureValidCoordinatesRange = true,
};
```

*Флаг `EnsureValidCoordinatesRange = true` указывает Aspose.GIS **проверять диапазон координат** для каждой добавляемой функции.*

### Шаг 3: Создать слой с сеткой
Теперь создаём новый слой внутри набора данных, применяя только что определённые параметры сетки. Мы будем использовать систему пространственных ссылок WGS84.

```csharp
using (var layer = dataset.CreateLayer("layer_name", options, SpatialReferenceSystem.Wgs84))
{
```

### Шаг 4: Добавить объекты в слой
Мы создаём два точечных объекта. Первая точка находится внутри сетки, а вторая намеренно выходит за её пределы, чтобы продемонстрировать **как обрабатывать ошибки выхода за диапазон**.

```csharp
var feature = layer.ConstructFeature();
feature.Geometry = new Point(10, 20) { M = 10.1282 };
layer.Add(feature);
feature = layer.ConstructFeature();
feature.Geometry = new Point(-410, 0) { M = 20.2343 };
```

### Шаг 5: Обработать исключения при добавлении объектов за пределами диапазона
Попытка добавить второй объект вызовет исключение, потому что его координата X (`-410`) находится за пределами определённой сетки. Мы перехватываем исключение и выводим понятное сообщение.

```csharp
try
{
    layer.Add(feature);
}
catch (GisException e)
{
    Console.WriteLine(e.Message); // X value -410 is out of valid range.
}
```

### Шаг 6: Очистка
Операторы `using` автоматически закрывают и освобождают набор данных и слой, гарантируя освобождение всех ресурсов.

## Распространённые проблемы и решения
| Проблема | Почему происходит | Решение |
|----------|-------------------|---------|
| **Исключение: «Значение X … выходит за допустимый диапазон».** | Координаты находятся за пределами точной сетки. | Отрегулируйте `XOrigin`, `YOrigin` или `XYScale`, чтобы охватить ваши данные, либо убедитесь, что входные данные находятся в заданном диапазоне. |
| **Объекты не отображаются в GIS‑просмотрщике** | Слой не сохранён или указан неверный пространственный справочник. | Проверьте, что `SpatialReferenceSystem.Wgs84` соответствует CRS просмотрщика, и что `Dataset.Create` выполнен успешно. |
| **Значения M игнорируются** | `MScale` установлен в 0 или слишком мал. | Установите разумный `MScale` (например, `1e4`), чтобы сохранять измерительные значения. |

## Советы по устранению неполадок
- **Тщательно проверяйте границы сетки** перед загрузкой больших пакетов данных; небольшая опечатка в `XOrigin` может привести к отклонению множества строк.  
- **Записывайте сообщение исключения** (как показано в блоке try‑catch) в файл при обработке автоматических импортов; это облегчает выявление закономерностей в данных, выходящих за диапазон.  
- **Используйте `EnsureValidCoordinatesRange = false` только для надёжных источников данных** — отключение проверки может привести к повреждённым геометриям.
