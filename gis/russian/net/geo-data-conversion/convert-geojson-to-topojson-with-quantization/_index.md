# Преобразование GeoJSON в TopoJSON с квантизацией

## Введение
Если вам нужно **преобразовать GeoJSON в TopoJSON** для веб‑картографии, мобильных GIS‑приложений или сценариев сжатия данных, вы попали в нужное место. В этом руководстве мы пошагово покажем, как преобразовать файл GeoJSON в компактный файл TopoJSON **с квантизацией**, используя библиотеку Aspose.GIS для .NET. Квантизация значительно уменьшает размер результата, сохраняя географическую точность, необходимую для точных визуализаций. Этот метод также помогает **уменьшить размер файла GeoJSON** и **сжать GIS‑данные** без потери качества.

## Быстрые ответы
- **Что делает квантизация?** Она уменьшает точность координат до фиксированного количества целочисленных шагов, сокращая размер файла без заметной потери деталей.  
- **Почему выбирать Aspose.GIS для этого преобразования?** Он предоставляет API в одну строку, полную поддержку .NET и встроенные параметры TopoJSON.  
- **Нужна ли лицензия?** Бесплатная пробная версия подходит для разработки; коммерческая лицензия требуется для продакшна.  
- **Какие версии .NET поддерживаются?** .NET Framework 4.5+, .NET Core 3.1+, .NET 5/6/7+.  
- **Сколько времени занимает преобразование?** Обычно менее секунды для файлов размером до нескольких мегабайт.

## Что такое преобразование GeoJSON в TopoJSON?
GeoJSON хранит геометрию каждой функции в отдельном списке координат, что может быть избыточным. TopoJSON, наоборот, кодирует общие линейные сегменты один раз, создавая **более компактное представление** — особенно полезно, когда необходимо обслуживать карты при ограниченной пропускной способности.

## Почему использовать конвертацию Aspose.GIS для GeoJSON → TopoJSON?
- **Конвертация одной функцией** – не требуется вручную разбирать или восстанавливать геометрии.  
- **Встроенная квантизация** – контролируйте размер вывода с помощью свойства `QuantizationNumber`.  
- **Кросс‑платформенный** – работает на Windows, Linux и macOS в средах .NET.  
- **Широкая поддержка форматов** – помимо GeoJSON/TopoJSON, Aspose.GIS работает с Shapefile, KML, GML и другими.  
- **aspose gis conversion**, который надёжно уменьшает размер файлов, сохраняя пространственную точность.

## Как преобразовать GeoJSON в TopoJSON с квантизацией?
Ниже представлено пошаговое руководство. Каждый шаг включает короткое объяснение и точный код, который нужно скопировать.

### Шаг 1: Определите пути и файл вывода
Установите путь к входному файлу GeoJSON и путь к целевому файлу TopoJSON. При необходимости скорректируйте расположения папок под структуру вашего проекта.

```csharp
string SampleGeoJsonPath = "Your Document Directory" + "sample.geojson";
var outputFilePath = "Your Document Directory" + "convertedSampleWithQuantization_out.topojson";
```

### Шаг 2: Укажите параметры конвертации (квантизация)
Настройте `ConversionOptions`, чтобы драйвер TopoJSON знал, насколько агрессивно квантизировать координаты. Чем выше `QuantizationNumber`, тем более детализированным будет результат, но размер файлов увеличится.

```csharp
var options = new ConversionOptions
{
    DestinationDriverOptions = new TopoJsonOptions
    {
        QuantizationNumber = 100_000,
    }
};
```

### Шаг 3: Выполните конвертацию
Вызовите статический метод `Convert` у `VectorLayer`. Эта одна строка считывает GeoJSON, применяет квантизацию и записывает файл TopoJSON.

```csharp
VectorLayer.Convert(SampleGeoJsonPath, Drivers.GeoJson, outputFilePath, Drivers.TopoJson, options);
```

Использование Aspose.GIS для **преобразования geojson в topojson** с квантизацией даёт лёгкий, готовый к веб‑использованию файл, который быстрее загружается в браузерах и на мобильных устройствах. Это также помогает соответствовать ограничениям пропускной способности в облачных GIS‑службах, делая решение более экономичным.

## Распространённые проблемы и их устранение
| Симптом | Вероятная причина | Решение |
|---------|-------------------|---------|
| **Файл вывода пустой** | Неправильный путь к файлу или отсутствие прав на чтение | Убедитесь, что `SampleGeoJsonPath` указывает на действительный файл и процесс имеет права чтения/записи. |
| **Топологические ошибки после конвертации** | Входной GeoJSON содержит недопустимые геометрии (например, самопересекающиеся полигоны) | Очистите GeoJSON с помощью GIS‑редактора или выполните проверку `Geometry.IsValid` перед конвертацией. |
| **Квантизация слишком агрессивна (визуальные искажения)** | `QuantizationNumber` установлен слишком низко | Увеличьте значение (например, с 50 000 до 100 000), чтобы сохранить большую точность. |
