# точка внутри полигона c# – Проверка Geometry Contains Another

## Введение
Если вы работаете над проектами **геопространственный анализ .net**, одной из наиболее распространённых задач является определение, находящееся в конкретном месте (точке) внутри заданной области (полигона). В этом руководстве мы покажем вам шаг за шагом, как выполнить проверку проверки **точки внутри полигона C#** с помощью библиотеки **Aspose.GIS .NET**. Независимо от того, создаёте ли вы картографическое приложение, сервис, основанный на размещении, или любое решение, требующее логики пространственного включения, приведенные ниже фрагменты кода помогут вам начать работу за считанные минуты.

## Быстрые ответы
- **Что означает «точка внутри многоугольника C#»?** Это пространственный запрос, который возвращает «истину», когда геометрия точек полностью находится внутри геометрии многоугольника.
- **Какая библиотека реализует это в .NET?** Aspose.GIS для .NET предоставляет методы `SpatiallyContains` и `Within`.
- **Нужна ли лицензия?** Доступна бесплатная пробная версия; для использования в продаже требуется коммерческая лицензия.
- **Совместима ли она с .NET Core / .NET 6+?** Да — Aspose.GIS полностью поддерживает современную среду выполнения .NET.
- **Сколько времени занимает производство?** Около 10 минут, чтобы скопировать код и запустить пример.

## Что такое точка внутри многоугольника C#?
Тест *точка внутри многоугольника* теперь найдите координаты объекта «Точка» внутри границ объекта «Многоугольник». В C# это обычно реализуется через библиотеку модулей, используя алгоритмы **Ray Casting** или **Winding Number**. Aspose.GIS абстрагирует эти конструкции и предлагает простой API: `polygon.SpatiallyContains(point)`.

## Почему использование Aspose.GIS .NET для геометрии содержит проверки точек?
- **Модель с богатой геометрией** – Поддерживает полигоны, мультиполигоны, линейные кольца и многое другое.
- **Высокопроизводительные пространственные операции** – Оптимизировано для больших наборов данных.
- **Кроссплатформенность** – Работает на .NET Framework, .NET Core и .NET5/6+.
- **Комплексная документация** – Примеры для выполнения **геопространственного анализа .net**.

## Распространенные случаи использования точки внутри многоугольника C#
- **Geofencing**: запуск действий, когда устройство находится в заранее определенной зоне или выходит из нее.
- **Визуализация карты**: Выделение регионов, содержащих выбранную вами точку.
- **Пространственная аналитика**: Фильтрация набора данных, оставляющая только записи, попадающие в заданную область.
- **Маршрут доставки**: Проверка того, что адрес доставки находится в зоне обслуживания.

В вашем C#‑проекте импортируйте необходимые пространства имён для использования функций Aspose.GIS:

using Aspose.Gis.Geometries;

## Шаг 1: Определение геометрических объектов
Сначала определите объекты геометрии с помощью классов Aspose.GIS. Здесь мы создаём полигон с внешним кольцом и внутренним кольцом (отверстием), а затем точку, которую будем проверять на включение.
```csharp
var geometry1 = new Polygon();
geometry1.ExteriorRing = new LinearRing(new[]
{
    new Point(0, 0),
    new Point(0, 4),
    new Point(4, 4),
    new Point(4, 0),
    new Point(0, 0),
});
geometry1.AddInteriorRing(new LinearRing(new[]
{
    new Point(1, 1),
    new Point(1, 3),
    new Point(3, 3),
    new Point(3, 1),
    new Point(1, 1),
}));
var geometry2 = new Point(2, 2);
```

## Шаг 2: Проверка пространственной привязки
Далее проверяем, содержит ли полигон **geometry1** точку **geometry2**. Метод `SpatiallyContains` возвращает `false`, потому что точка находится внутри внутреннего кольца (отверстия).
```csharp
Console.WriteLine(geometry1.SpatiallyContains(geometry2)); // False
```

## Шаг 3: Определение другой геометрии
Теперь определяем вторую точку, которая находится во внешнем кольце, но вне внутреннего кольца.
```csharp
var geometry3 = new Point(0.5, 0.5);
```

## Шаг 4: Повторная проверка пространственной привязки
Выполнение той же проверки включения с новой точкой возвращает `true`, подтверждая, что точка действительно находится внутри внешней границы полигона.
```csharp
Console.WriteLine(geometry1.SpatiallyContains(geometry3)); // True
```

## Шаг 5: Эквивалентная функциональность
Aspose.GIS также предоставляет обратный метод `Within`. Следующая строка демонстрирует, что `geometry3.Within(geometry1)` даёт тот же результат, что и `geometry1.SpatiallyContains(geometry3)`.
```csharp
Console.WriteLine(geometry3.Within(geometry1)); // True
```

## Распространенные проблемы и решения
| Выпуск | Почему это происходит | Исправить |
|-------|----------------|-----|
| **Неожиданный «ложный» результат** | Точка находится внутри отверстий (внутреннего кольца) полигона. | Убедитесь, что теперь правильный многоугольник, или используйте «geometry1.ExteriorRing» для простых многоугольников без отверстий. |
| **NullReferenceException** | Объекты определения не являются логическими перед вызовом `SpatiallyContains`. | создайте оба объекта — полигон и точку — перед вызовом пространственных методов. |
| **Снижение производительности при работе с большими наборами данных** | Повторное создание объектов оценки внутри циклов. | Переиспользуйте экземпляры геометрии или обрабатывайте пакеты с помощью GeometryCollection. |
