# वेक्टर लेयर बनाएं, Aspose.GIS for .NET के साथ प्रिसीजन सीमित करें

## परिचय
जब आप जियोस्पेशियल डेटा के साथ काम करते हैं, तो अक्सर आपको **वेक्टर लेयर बनाना** आवश्यक होता है और यह तय करना पड़ता है कि आपको निर्देशांक विवरण के कितने दशमलव स्थानों की वास्तव में आवश्यकता है। प्रिसीजन को सीमित करने से न केवल प्रोसेसिंग तेज़ होती है बल्कि **शेपफ़ाइल आकार घटाएँ** भी संभव है, जिससे संग्रहण और ट्रांसफ़र अधिक कुशल बनता है। इस ट्यूटोरियल में हम वेक्टर लेयर बनाने, एक सरल पॉइंट ज्योमेट्री लिखने, और फिर इसे सटीक और गोलाई वाले प्रिसीजन मॉडल दोनों का उपयोग करके पढ़ने की प्रक्रिया को देखेंगे। अंत तक आप समझ जाएंगे कि **प्रिसीजन मॉडल सेट करें** विकल्प कैसे आपके एप्लिकेशन की सटीकता आवश्यकताओं के अनुरूप होते हैं।

## त्वरित उत्तर
- **“limit precision” का क्या अर्थ है?** यह निर्देशांक मानों को परिभाषित दशमलव स्थानों तक गोल करता है।  
- **पहले वेक्टर लेयर क्यों बनाएं?** वेक्टर लेयर वह कंटेनर है जो बिंदु, रेखा और बहुभुज जैसी ज्यामितियों को संग्रहीत करता है।  
- **कौन से प्रिसीजन मॉडल उपलब्ध हैं?** `PrecisionModel.Exact` (कोई गोलाई नहीं) और `PrecisionModel.Rounding(n)` (*n* दशमलव तक गोल करता है)।  
- **क्या इसे आज़माने के लिए लाइसेंस चाहिए?** एक मुफ्त ट्रायल रिलीज़ पेज से उपलब्ध है।  
- **कौन से .NET संस्करण समर्थित हैं?** .NET Framework 4.5+, .NET Core, और .NET 5/6+।

## प्रिसीजन सीमित करने का कारण और यह कैसे मदद करता है?
- **प्रदर्शन वृद्धि** – कम अंक मतलब कम डेटा को पार्स और सीरियलाइज़ करना।  
- **छोटी फ़ाइलें** – निर्देशांक को गोल करने से शैपफ़ाइल का आकार उल्लेखनीय रूप से घट सकता है, विशेषकर बड़े डेटासेट के लिए।  
- **पर्याप्त सटीकता** – कई GIS विश्लेषणों को उप‑मिलीमीटर प्रिसीजन की आवश्यकता नहीं होती, इसलिए 2‑3 दशमलव तक गोल करना अक्सर पर्याप्त होता है।

## पूर्वापेक्षाएँ
इस यात्रा पर निकलने से पहले, सुनिश्चित करें कि आपके पास निम्नलिखित पूर्वापेक्षाएँ मौजूद हैं:
1. **इंस्टॉलेशन** – Aspose.GIS for .NET लाइब्रेरी आपके विकास वातावरण में स्थापित होनी चाहिए। यदि नहीं, तो आप इसे [releases page](https://releases.aspose.com/gis/net/) से डाउनलोड कर सकते हैं।
2. **.NET से परिचितता** – प्रदान किए गए कोड उदाहरणों को समझने और लागू करने के लिए C# और .NET फ्रेमवर्क का मूल ज्ञान आवश्यक है।
3. **विकास वातावरण** – Visual Studio जैसे कार्यशील .NET विकास वातावरण की आवश्यकता है।
4. **डॉक्यूमेंट डायरेक्टरी** – ऐसी डायरेक्टरी स्थापित रखें जहाँ आप प्रक्रिया के दौरान उत्पन्न शैपफ़ाइल को संग्रहीत और एक्सेस कर सकें।

## नेमस्पेस इम्पोर्ट करें
ज्यामितियों को पढ़ते समय प्रिसीजन सीमित करने की कार्यक्षमता लागू करने से पहले, आइए आवश्यक नेमस्पेस इम्पोर्ट करना सुनिश्चित करें:
```csharp
using Aspose.Gis;
using Aspose.Gis.Formats.Shapefile;
using Aspose.Gis.Geometries;
using Aspose.GIS.Examples.CSharp;
using System;
using System.Collections.Generic;
using System.Linq;
using System.Text;
using System.Threading.Tasks;
```

## वेक्टर लेयर कैसे बनाएं
पहला कदम **वेक्टर लेयर बनाना** है जो हमारी ज्यामिति को रखेगा। यह लेयर एक शैपफ़ाइल के रूप में सहेजा जाएगा ताकि हम बाद में इसे विभिन्न प्रिसीजन सेटिंग्स के साथ पुनः खोल सकें।
```csharp
string path = "Your Document Directory" + "LimitPrecisionWhenReadingGeometries_out.shp";
using (VectorLayer layer = VectorLayer.Create(path, Drivers.Shapefile))
{
	var feature = layer.ConstructFeature();
	feature.Geometry = new Point(1.10234, 2.09743);
	layer.Add(feature);
}
```

## प्रिसीजन विकल्प सेट करना
अगला, हमें ज्यामितियों को पढ़ने के विकल्प परिभाषित करने की आवश्यकता है, वांछित प्रिसीजन मॉडल निर्दिष्ट करते हुए। हम सटीक प्रिसीजन से शुरू कर सकते हैं:
```csharp
var options = new ShapefileOptions();
// read data as‑is.
options.XYPrecisionModel = PrecisionModel.Exact;
```

## सटीक प्रिसीजन के साथ ज्यामितियों को पढ़ना
अब, निर्दिष्ट विकल्पों के साथ वेक्टर लेयर खोलें ताकि सटीक प्रिसीजन के साथ ज्यामितियों को पढ़ा जा सके:
```csharp
using (VectorLayer layer = VectorLayer.Open(path, Drivers.Shapefile, options))
{
	var point = (IPoint)layer[0].Geometry;
	// 1.10234, 2.09743
	Console.WriteLine("{0}, {1}", point.X, point.Y);
}
```

## प्रिसीजन को ट्रंकेट करना
यदि हम प्रिसीजन को विशिष्ट दशमलव स्थानों तक ट्रंकेट करना चाहते हैं, तो हम प्रिसीजन मॉडल को उसी अनुसार समायोजित कर सकते हैं:
```csharp
options.XYPrecisionModel = PrecisionModel.Rounding(2);
using (VectorLayer layer = VectorLayer.Open(path, Drivers.Shapefile, options))
{
	var point = (IPoint)layer[0].Geometry;
	// 1.1, 2.1
	Console.WriteLine("{0}, {1}", point.X, point.Y);
}
```

## विभिन्न परिदृश्यों के लिए प्रिसीजन मॉडल कैसे सेट करें
आप सोच सकते हैं कि `Exact` बनाम `Rounding` कब उपयोग करें। यहाँ दो सामान्य परिदृश्य हैं:

| परिदृश्य | सिफारिश किया गया मॉडल | कारण |
|----------|-------------------|--------|
| उच्च‑सटीकता वैज्ञानिक विश्लेषण | `PrecisionModel.Exact` | कोऑर्डिनेट विवरण का कोई नुकसान नहीं |
| वेब‑मैपिंग टाइल्स या मोबाइल ऐप्स | `PrecisionModel.Rounding(2)` | फ़ाइल आकार घटाता है और रेंडरिंग को तेज़ करता है |

सही मॉडल चुनना **प्रिसीजन मॉडल सेट** करने की निर्णय‑प्रक्रिया का हिस्सा है जो सटीकता को प्रदर्शन के साथ संतुलित करता है।

## सामान्य समस्याएँ और समाधान
- **अप्रत्याशित कोऑर्डिनेट मान** – लेयर खोलने से *पहले* `options.XYPrecisionModel` सेट करना सुनिश्चित करें। खोलने के बाद इसे बदलने से कोई प्रभाव नहीं पड़ता।  
- **फ़ाइल नहीं मिली** – सुनिश्चित करें कि `path` वेरिएबल एक वैध डायरेक्टरी की ओर इशारा करता है और शैपफ़ाइल पिछले चरण में सफलतापूर्वक बनाई गई थी।  
- **गलत ज्यामिति प्रकार** – उदाहरण में `Point` का उपयोग किया गया है। अन्य ज्यामिति प्रकारों (जैसे `LineString`) के लिए, कास्टिंग वास्तविक प्रकार से मेल खानी चाहिए।  

## शैपफ़ाइल आकार घटाने के टिप्स
- `PrecisionModel.Rounding` का उपयोग सबसे कम दशमलव संख्या के साथ करें जो आपकी सटीकता आवश्यकताओं को पूरा करता हो।  
- लेयर लिखने से पहले अनावश्यक एट्रिब्यूट फ़ील्ड हटाएँ।  
- यदि आपको फ़ाइलें ट्रांसफ़र करनी हैं तो परिणामस्वरूप `.shp`, `.shx`, और `.dbf` फ़ाइलों को मानक ZIP यूटिलिटीज़ से संपीड़ित करें।

## निष्कर्ष
संक्षेप में, ज्यामितियों को पढ़ते समय प्रिसीजन का प्रबंधन जियोस्पेशियल डेटा हेरफेर का एक महत्वपूर्ण पहलू है। Aspose.GIS for .NET इसको कुशलता से हासिल करने के लिए मजबूत कार्यात्मकताएँ प्रदान करता है। ऊपर दिए गए चरणों का पालन करके आप सहजता से **वेक्टर लेयर बनाएं** ऑब्जेक्ट्स, **प्रिसीजन मॉडल सेट करें**, और आवश्यकतानुसार **शेपफ़ाइल आकार घटाएँ**, जिससे आपके एप्लिकेशन में डेटा हैंडलिंग इष्टतम बनती है।
