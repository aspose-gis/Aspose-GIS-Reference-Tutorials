# वेक्टर लेयर बनाएं और लेयर स्पैशियल रेफ़रेंस सिस्टम सेट करें

## परिचय
भौगोलिक सूचना प्रणाली (GIS) के विशाल परिदृश्य में, **Aspose.GIS for .NET** डेवलपर्स के लिए एक मजबूत और बहुमुखी टूल के रूप में उभरता है। इस ट्यूटोरियल में आप **वेक्टर लेयर बनाएँगे**, उसका स्पैशियल रेफ़रेंस निर्धारित करेंगे, एक पॉइंट फीचर जोड़ेंगे, और EPSG कोड प्राप्त करेंगे—सभी स्पष्ट, चरण‑दर‑चरण मार्गदर्शन के साथ। चाहे आप एक अनुभवी GIS इंजीनियर हों या अभी शुरुआत कर रहे हों, ये तकनीकें आपको शैपफ़ाइल कोऑर्डिनेट सिस्टम सही तरीके से सेट करने और आपके स्पैशियल डेटा वर्कफ़्लो की विश्वसनीयता बढ़ाने में मदद करेंगी।

## त्वरित उत्तर
- **“वेक्टर लेयर बनाना” का क्या अर्थ है?** यह एक नई GIS लेयर (जैसे Shapefile) बनाता है जो पॉइंट, लाइन या पॉलीगॉन जैसी ज्योमेट्रीज़ संग्रहीत कर सकती है।  
- **उदाहरण में कौन सा EPSG कोड उपयोग किया गया है?** EPSG 26918 (NAD83 / UTM ज़ोन 18N)।  
- **लेयर बनाने के बाद क्या मैं पॉइंट फीचर जोड़ सकता हूँ?** हाँ—`ConstructFeature()` का उपयोग करें और एक `Point` ज्योमेट्री असाइन करें।  
- **लेयर का CRS कैसे प्राप्त करें?** `layer.SpatialReferenceSystem.EpsgCode` या `.Name` पढ़ें।  
- **क्या Aspose.GIS के लिए लाइसेंस आवश्यक है?** एक फ्री ट्रायल उपलब्ध है; प्रोडक्शन उपयोग के लिए लाइसेंस आवश्यक है।

## पूर्वापेक्षाएँ
ट्यूटोरियल शुरू करने से पहले सुनिश्चित करें कि आपके पास निम्नलिखित पूर्वापेक्षाएँ हों:
- .NET प्रोग्रामिंग का कार्यात्मक ज्ञान।  
- आपके सिस्टम पर Visual Studio स्थापित हो।  
- Aspose.GIS for .NET लाइब्रेरी, जिसे आप [यहाँ](https://releases.aspose.com/gis/net/) से डाउनलोड कर सकते हैं।  
- GIS में स्पैशियल रेफ़रेंस सिस्टम्स की बुनियादी समझ।

## नेमस्पेसेस इम्पोर्ट करें
अपने .NET प्रोजेक्ट में, Aspose.GIS द्वारा प्रदान की गई कार्यक्षमताओं तक पहुँचने के लिए आवश्यक नेमस्पेसेस इम्पोर्ट करें। नीचे दिया गया कोड स्निपेट उपयोग करें:

```csharp
using Aspose.Gis;
using Aspose.Gis.Geometries;
using Aspose.Gis.SpatialReferencing;
using System;
```

## चरण 1: डॉक्यूमेंट डायरेक्टरी निर्दिष्ट करें
सबसे पहले अपने डॉक्यूमेंट डायरेक्टरी का पाथ निर्दिष्ट करें। यह वह स्थान होगा जहाँ आपके स्पैशियल डेटा फ़ाइलें संग्रहीत होंगी।

```csharp
string dataDir = "Your Document Directory";
```

## चरण 2: स्पैशियल रेफ़रेंस निर्धारित करें और शैपफ़ाइल कोऑर्डिनेट सिस्टम सेट करें
Shapefile के पाथ को निर्धारित करें, और **स्पैशियल रेफ़रेंस** EPSG कोड (इस उदाहरण में 26918) का उपयोग करके परिभाषित करें। यह चरण **लेयर के लिए शैपफ़ाइल कोऑर्डिनेट सिस्टम सेट करता है**।

```csharp
string path = dataDir + "SpecifyLayerSpatialReference_out.shp";
var srs = SpatialReferenceSystem.CreateFromEpsg(26918);
```

## चरण 3: वेक्टर लेयर बनाएं
अब **वेक्टर लेयर बनाएं** निर्दिष्ट Shapefile पाथ, ड्राइवर टाइप (Shapefile), और अभी परिभाषित स्पैशियल रेफ़रेंस सिस्टम के साथ।

```csharp
using (VectorLayer layer = VectorLayer.Create(path, Drivers.Shapefile, srs))
{
    // Your code for further operations on the layer goes here
}
```

## चरण 4: लेयर में पॉइंट फीचर जोड़ें
एक नया फीचर बनाएं और **पॉइंट फीचर** को उसकी ज्योमेट्री (कोऑर्डिनेट 60, 24 वाला `Point`) सेट करके जोड़ें। फिर इस फीचर को वेक्टर लेयर में जोड़ें।

```csharp
var feature = layer.ConstructFeature();
feature.Geometry = new Point(60, 24);
layer.Add(feature);
```

## चरण 5: स्पैशियल रेफ़रेंस सिस्टम जानकारी प्राप्त करें (EPSG कोड प्राप्त करें)
वेक्टर लेयर खोलें और स्पैशियल रेफ़रेंस सिस्टम की जानकारी प्राप्त करें, जैसे EPSG कोड और मानव‑पठनीय नाम। यह दर्शाता है कि **EPSG कोड कैसे प्राप्त करें** और **लेयर का CRS कैसे सेट करें**।

```csharp
using (VectorLayer layer = VectorLayer.Open(path, Drivers.Shapefile))
{
    Console.WriteLine(layer.SpatialReferenceSystem.EpsgCode); // 26918
    Console.WriteLine(layer.SpatialReferenceSystem.Name);     // NAD83_UTM_zone_18N
}
```

इन चरणों को अपनी विशिष्ट उपयोग केस के अनुसार दोहराएँ, और आप **वेक्टर लेयर बनाना** और Aspose.GIS for .NET के साथ स्पैशियल रेफ़रेंस को प्रबंधित करने में निपुण हो जाएंगे।

## सामान्य समस्याएँ और समाधान
| समस्या | क्यों होती है | समाधान |
|-------|----------------|-----|
| **लेयर खोलने में विफल** | गलत ड्राइवर या फ़ाइल पाथ भ्रष्ट | `Drivers.Shapefile` की जाँच करें और सुनिश्चित करें कि `path` मौजूदा `.shp` फ़ाइल की ओर इशारा कर रहा है। |
| **गलत CRS प्रदर्शित** | गलत EPSG कोड उपयोग किया गया | विश्वसनीय स्रोत (जैसे EPSG.io) से EPSG कोड दोबारा जांचें। |
| **फ़ीचर सेव नहीं हुआ** | `using` ब्लॉक के भीतर `layer.Add(feature)` नहीं बुलाया गया | लेयर डिस्पोज़ होने से पहले `Add` मेथड को निष्पादित करना सुनिश्चित करें। |

## अतिरिक्त अक्सर पूछे जाने वाले प्रश्न
**प्र.: मौजूदा Shapefile का CRS कैसे बदलें?**  
उ.: लेयर खोलें, इच्छित EPSG कोड के साथ नया `SpatialReferenceSystem` बनाएं, और उसे `layer.SpatialReferenceSystem` को असाइन करें, फिर सहेजें।

**प्र.: क्या मैं उसी तरीके से अन्य ज्योमेट्री प्रकार (जैसे polygons) जोड़ सकता हूँ?**  
उ.: हाँ—`new Point(x, y)` को `new Polygon(...)` या `new LineString(...)` से बदलें।

**प्र.: बड़े डेटा सेट्स के साथ कुशलता से काम करना संभव है?**  
उ.: स्ट्रीमिंग API (`VectorLayer.Create` के साथ `FeatureCollection`) का उपयोग करें और संसाधन मुक्त करने के लिए लेयर्स को तुरंत डिस्पोज़ करें।

**प्र.: क्या Aspose.GIS कोऑर्डिनेट ट्रांसफ़ॉर्मेशन को सपोर्ट करता है?**  
उ.: हाँ—`Geometry.Transform(targetSrs)` का उपयोग करके विभिन्न स्पैशियल रेफ़रेंस के बीच ज्योमेट्रीज़ को री‑प्रोजेक्ट किया जा सकता है।

**प्र.: कौन‑से .NET संस्करण समर्थित हैं?**  
उ.: Aspose.GIS .NET Framework 4.6+, .NET Core 3.1+, .NET 5/6/7 के साथ काम करता है।

इस ट्यूटोरियल में हमने **वेक्टर लेयर बनाना**, उसका स्पैशियल रेफ़रेंस निर्धारित करना, **पॉइंट फीचर जोड़ना**, और Aspose.GIS for .NET का उपयोग करके **EPSG कोड प्राप्त करना** सीखा। इन चरणों में निपुण होकर आप शैपफ़ाइल कोऑर्डिनेट सिस्टम सही ढंग से सेट कर सकते हैं, लेयर CRS प्रबंधित कर सकते हैं, और विश्वसनीय GIS एप्लिकेशन बना सकते हैं।
