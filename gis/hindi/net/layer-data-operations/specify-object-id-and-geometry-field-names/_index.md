# पॉइंट फीचर जोड़ें और ऑब्जेक्ट आईडी तथा जियोमेट्री फ़ील्ड नाम निर्दिष्ट करें

## परिचय
Geographic Information Systems (GIS) के क्षेत्र में Aspose.GIS for .NET का उपयोग करके यात्रा शुरू करना डेवलपर्स और उत्साही लोगों दोनों के लिए संभावनाओं की एक नई दुनिया खोलता है। इस ट्यूटोरियल में, **आप सीखेंगे कि पॉइंट फीचर कैसे जोड़ें** साथ ही **फ़ील्ड नाम कैसे सेट करें** और **ऑब्जेक्ट आईडी** मान कैसे पढ़ें, जिससे आपके स्पैशियल डेटा पर पूर्ण नियंत्रण मिलेगा।

## त्वरित उत्तर
- **इस गाइड का मुख्य उद्देश्य क्या है?** पॉइंट फीचर जोड़ने और ऑब्जेक्ट आईडी तथा जियोमेट्री फ़ील्ड नाम कॉन्फ़िगर करने का तरीका दिखाना।  
- **कौनसी लाइब्रेरी उपयोग की गई है?** Aspose.GIS for .NET।  
- **क्या मुझे लाइसेंस चाहिए?** एक फ्री ट्रायल उपलब्ध है; प्रोडक्शन के लिए लाइसेंस आवश्यक है।  
- **कौनसे .NET संस्करण समर्थित हैं?** .NET Framework 4.5+, .NET Core 3.1+, .NET 5/6+।  
- **इंसर्शन के बाद क्या मैं ऑब्जेक्ट आईडी पढ़ सकता हूँ?** हाँ – ट्यूटोरियल दर्शाता है कि लेयर से ऑब्जेक्ट आईडी कैसे पढ़ें।

## पूर्वापेक्षाएँ
ट्यूटोरियल शुरू करने से पहले सुनिश्चित करें कि आपके पास निम्नलिखित पूर्वापेक्षाएँ हों:
- Aspose.GIS for .NET: लाइब्रेरी को [यहाँ](https://releases.aspose.com/gis/net/) से डाउनलोड और इंस्टॉल करें।  
- डॉक्यूमेंट डायरेक्टरी: अपने दस्तावेज़ों के लिए एक डायरेक्टरी सेट करें जहाँ आप जियोडेटाबेस स्टोर करेंगे।  
- .NET वातावरण: सुनिश्चित करें कि आपके पास एक कार्यशील .NET वातावरण है।

## नेमस्पेस इम्पोर्ट करें
शुरू करने के लिए, आपको अपने प्रोजेक्ट में आवश्यक नेमस्पेस इम्पोर्ट करने होंगे। ये नेमस्पेस Aspose.GIS for .NET के साथ इंटरैक्ट करने के लिए आवश्यक क्लास और मेथड प्रदान करते हैं।

```csharp
using Aspose.Gis;
using Aspose.Gis.Formats.FileGdb;
using Aspose.Gis.Geometries;
using System;
using Aspose.Gis.SpatialReferencing;
```

## चरण 1: पॉइंट फीचर जोड़ें और ऑब्जेक्ट आईडी तथा जियोमेट्री फ़ील्ड नाम निर्दिष्ट करें
इस चरण में, आप अपने GIS डेटा के लिए ऑब्जेक्ट आईडी और जियोमेट्री फ़ील्ड नाम कैसे सेट करें, सीखेंगे। यह कुशल डेटा प्रबंधन और बाद में **ऑब्जेक्ट आईडी पढ़ने** के लिए महत्वपूर्ण है।

### चरण 1.1: डॉक्यूमेंट डायरेक्टरी सेट करें
अपने डॉक्यूमेंट डायरेक्टरी का पाथ परिभाषित करके शुरू करें:

```csharp
string dataDir = "Your Document Directory";
```

### चरण 1.2: GeoDatabase बनाएं और विकल्प निर्धारित करें
ऑब्जेक्ट आईडी और जियोमेट्री के लिए इच्छित **फ़ील्ड नाम सेट** करके एक GeoDatabase बनाएं:

```csharp
var path = dataDir + "NamesOfObjectIdAndGeometryFields_out.gdb";
using (var dataset = Dataset.Create(path, Drivers.FileGdb))
{
    var options = new FileGdbOptions
    {
        ObjectIdFieldName = "OID",         // Specify the Object ID field name
        GeometryFieldName = "POINT",       // Specify the Geometry field name
    };
```

### चरण 1.3: लेयर बनाएं और जोड़ें
GeoDatabase के भीतर एक लेयर बनाएं और एक फीचर जोड़ें जो पॉइंट जियोमेट्री का प्रतिनिधित्व करता है:

```csharp
using (var layer = dataset.CreateLayer("layer_name", options, SpatialReferenceSystem.Wgs84))
{
    var feature = layer.ConstructFeature();
    feature.Geometry = new Point(12.32, 34.21);  // Specify the geometry (in this case, a point)
    layer.Add(feature);
}
```

### चरण 1.4: लेयर खोलें और डेटा प्राप्त करें
लेयर खोलें और वह फीचर प्राप्त करें जिसे आपने अभी जोड़ा था। यह दर्शाता है कि डेटासेट से **ऑब्जेक्ट आईडी** कैसे पढ़ें:

```csharp
using (var layer = dataset.OpenLayer("layer_name"))
{
    var feature = layer[0];
    Console.WriteLine(feature.GetValue<int>("OID")); // Output: 1
}
```

## कस्टम फ़ील्ड नाम क्यों सेट करें?
ऑब्जेक्ट आईडी और जियोमेट्री फ़ील्ड नाम को कस्टमाइज़ करने से आप मौजूदा डेटाबेस के साथ इंटीग्रेशन या डाउनस्ट्रीम एप्लिकेशन द्वारा आवश्यक नामकरण मानकों का पालन करने में लचीलापन प्राप्त करते हैं। यह डेटा को अधिक स्व-व्याख्यात्मक बनाता है, जिससे रखरखाव और सहयोग आसान हो जाता है।

## सामान्य समस्याएँ और ट्रबलशूटिंग
- **डायरेक्टरी नहीं मिली** – सुनिश्चित करें कि `dataDir` किसी मौजूदा फ़ोल्डर की ओर इशारा कर रहा है; अन्यथा, `Dataset.Create` एक एक्सेप्शन फेंकेगा।  
- **फ़ील्ड नाम टकराव** – यदि जियोडेटाबेस में समान नाम वाला फ़ील्ड पहले से मौजूद है, तो निर्माण विफल हो जाएगा। अद्वितीय नाम चुनें।  
- **स्पैशियल रेफ़रेंस मिसमैच** – हमेशा स्पैशियल रेफ़रेंस सिस्टम (जैसे, WGS84) को उस डेटा के साथ मिलाएँ जिसे आप स्टोर करने जा रहे हैं।

## निष्कर्ष
बधाई हो! आपने सफलतापूर्वक **पॉइंट फीचर जोड़ना**, **फ़ील्ड नाम सेट करना**, और Aspose.GIS for .NET का उपयोग करके **ऑब्जेक्ट आईडी पढ़ना** सीख लिया है। यह आधार आपको मजबूत GIS एप्लिकेशन बनाने और स्पैशियल डेटा को आत्मविश्वास के साथ प्रबंधित करने में सक्षम बनाता है।

## अतिरिक्त अक्सर पूछे जाने वाले प्रश्न

**प्रश्न:** क्या लेयर बन जाने के बाद मैं ऑब्जेक्ट आईडी फ़ील्ड बदल सकता हूँ?  
**उत्तर:** नहीं। फ़ील्ड नाम लेयर बनते समय निर्धारित होता है; इसे बदलने के लिए आपको एक नया `FileGdbOptions` ऑब्जेक्ट के साथ लेयर को पुनः बनाना होगा।

**प्रश्न:** ऑब्जेक्ट आईडी के अलावा अन्य एट्रिब्यूट मान कैसे प्राप्त करूँ?  
**उत्तर:** `feature.GetValue<T>("FieldName")` का उपयोग करें जहाँ `FieldName` वह एट्रिब्यूट का नाम है जिसे आप पढ़ना चाहते हैं।

**प्रश्न:** क्या एक ही बैच में कई पॉइंट फीचर जोड़ना संभव है?  
**उत्तर:** हाँ। अपने डेटा पर लूप करें, प्रत्येक पॉइंट के लिए एक फीचर बनाएं, उसकी जियोमेट्री सेट करें, और उसी `using` ब्लॉक के भीतर `layer.Add(feature)` कॉल करें।

**प्रश्न:** क्या Aspose.GIS अन्य जियोमेट्री प्रकारों का समर्थन करता है?  
**उत्तर:** बिल्कुल। आप `LineString`, `Polygon`, `MultiPoint` आदि के साथ काम कर सकते हैं, उपयुक्त जियोमेट्री ऑब्जेक्ट बनाकर।

**प्रश्न:** यदि मैं किसी गैर‑मौजूद ऑब्जेक्ट आईडी को पढ़ने की कोशिश करूँ तो क्या होगा?  
**उत्तर:** आउट‑ऑफ़‑रेंज इंडेक्स (जैसे, `layer[10]` जब केवल एक फीचर मौजूद हो) तक पहुँचने से `IndexOutOfRangeException` फेंका जाएगा। हमेशा पहले `layer.Count` जांचें।
