# फ़ाइल GDB लेयर के लिए Aspose.GIS में ग्रिड कैसे सेट करें

## परिचय
इस ट्यूटोरियल में, आप **फ़ाइल जियोडेटाबेस (GDB) लेयर** के लिए ग्रिड सेट करना सीखेंगे, Aspose.GIS for .NET का उपयोग करके। एक प्रिसीजन ग्रिड सेट करने से आप **कोऑर्डिनेट रेंज को वैधता** प्रदान कर सकते हैं, आउट‑ऑफ‑रेंज त्रुटियों को रोकते हैं, और यह सुनिश्चित करते हैं कि आप **लेयर में फीचर जोड़ते** समय डेटा सटीक और विश्वसनीय रहे। हम प्रत्येक चरण को विस्तार से बताएँगे, सेटिंग्स क्यों महत्वपूर्ण हैं समझाएँगे, और सामान्य समस्याओं को कैसे संभालें दिखाएँगे।

## त्वरित उत्तर
- **“ग्रिड सेट” का क्या मतलब है?** यह GIS लेयर के लिए कोऑर्डिनेट प्रिसीजन और वैध रेंज को परिभाषित करता है।  
- **प्रिसीजन ग्रिड क्यों उपयोग करें?** यह आपके डेटा को अमान्य कोऑर्डिनेट से बचाता है और स्टोरेज दक्षता बढ़ाता है।  
- **यह सुविधा कौनसी लाइब्रेरी प्रदान करती है?** Aspose.GIS for .NET।  
- **क्या लाइसेंस की आवश्यकता है?** एक ट्रायल उपलब्ध है; उत्पादन के लिए वाणिज्यिक लाइसेंस आवश्यक है।  
- **क्या इसे .NET Core के साथ उपयोग कर सकते हैं?** हाँ, Aspose.GIS .NET Framework और .NET Core दोनों को सपोर्ट करता है।

## प्रिसीजन ग्रिड क्या है और इसे क्यों सेट करें?
एक प्रिसीजन ग्रिड पैरामीटरों (ऑरिजिन, स्केल आदि) का सेट है जो GIS इंजन को बताता है कि कोऑर्डिनेट मानों को कैसे राउंड और स्टोर किया जाए। ग्रिड परिभाषित करके आप **कोऑर्डिनेट रेंज को वैधता** स्वचालित रूप से कर सकते हैं, और ग्रिड के बाहर कोई बिंदु डालने का प्रयास करने पर अपवाद उत्पन्न होगा—जिससे आप **आउट‑ऑफ‑रेंज** स्थितियों को विकास के शुरुआती चरण में संभाल सकते हैं।

## पूर्वापेक्षाएँ
शुरू करने से पहले सुनिश्चित करें कि आपके पास निम्नलिखित स्थापित हैं:

1. **Visual Studio** – कोई भी नवीनतम संस्करण (Community, Professional, या Enterprise)।  
2. **Aspose.GIS for .NET** – इसे [वेबसाइट](https://releases.aspose.com/gis/net/) से डाउनलोड करें।  
3. **बेसिक C# ज्ञान** – आपको .NET कंसोल प्रोजेक्ट बनाने में सहज होना चाहिए।

## नेमस्पेस इम्पोर्ट करें
Aspose.GIS के साथ काम करने के लिए आवश्यक नेमस्पेस इम्पोर्ट करें:

```csharp
using Aspose.Gis;
using Aspose.Gis.Formats.FileGdb;
using Aspose.Gis.Geometries;
using Aspose.Gis.SpatialReferencing;
using System;
using System.Text;
```

## फ़ाइल GDB लेयर में ग्रिड कैसे सेट करें
नीचे एक चरण‑दर‑चरण गाइड है जो दिखाता है कि ग्रिड को कैसे कॉन्फ़िगर करें, लेयर बनाएं, और सुरक्षित रूप से **लेयर में फीचर जोड़ें**।

### चरण 1: डेटासेट बनाएं
हम एक नया फ़ाइल जियोडेटाबेस डेटासेट बनाते हैं। यही वह जगह है जहाँ लेयर स्थित होगी।

```csharp
var path = "Your Document Directory" + "PrecisionGrid_out.gdb";
using (var dataset = Dataset.Create(path, Drivers.FileGdb))
{
```

### चरण 2: प्रिसीजन ग्रिड विकल्प निर्धारित करें
यहाँ हम ग्रिड पैरामीटर सेट करते हैं। अपने प्रोजेक्ट के कोऑर्डिनेट सिस्टम के अनुसार ऑरिजिन और स्केल को समायोजित करें।

```csharp
var options = new FileGdbOptions
{
    CoordinatePrecisionGrid = new FileGdbCoordinatePrecisionGrid
    {
        XOrigin = -400,
        YOrigin = -400,
        XYScale = 1e10,
        MOrigin = 0,
        MScale = 1e4,
    },
    EnsureValidCoordinatesRange = true,
};
```

*`EnsureValidCoordinatesRange = true` फ़्लैग Aspose.GIS को **कोऑर्डिनेट रेंज को वैधता** करने के लिए बताता है, प्रत्येक फीचर के जोड़ने पर।*

### चरण 3: ग्रिड के साथ लेयर बनाएं
अब हम डेटासेट के अंदर एक नई लेयर बनाते हैं, और अभी निर्धारित ग्रिड विकल्प लागू करते हैं। हम WGS84 स्पैशियल रेफ़रेंस सिस्टम का उपयोग करेंगे।

```csharp
using (var layer = dataset.CreateLayer("layer_name", options, SpatialReferenceSystem.Wgs84))
{
```

### चरण 4: लेयर में फीचर जोड़ें
हम दो पॉइंट फीचर बनाते हैं। पहला बिंदु ग्रिड के भीतर है, जबकि दूसरा जानबूझकर बाहर रखा गया है ताकि **आउट‑ऑफ‑रेंज** त्रुटियों को दिखाया जा सके।

```csharp
var feature = layer.ConstructFeature();
feature.Geometry = new Point(10, 20) { M = 10.1282 };
layer.Add(feature);
feature = layer.ConstructFeature();
feature.Geometry = new Point(-410, 0) { M = 20.2343 };
```

### चरण 5: आउट‑ऑफ‑रेंज फीचर जोड़ते समय अपवाद संभालें
दूसरे फीचर को जोड़ने का प्रयास करने पर अपवाद उत्पन्न होगा क्योंकि उसका X कोऑर्डिनेट (`-410`) परिभाषित ग्रिड से बाहर है। हम अपवाद को पकड़ते हैं और स्पष्ट संदेश प्रदर्शित करते हैं।

```csharp
try
{
    layer.Add(feature);
}
catch (GisException e)
{
    Console.WriteLine(e.Message); // X value -410 is out of valid range.
}
```

### चरण 6: सफ़ाई करें
`using` स्टेटमेंट्स स्वचालित रूप से डेटासेट और लेयर को बंद और डिस्पोज़ कर देते हैं, जिससे सभी संसाधन मुक्त हो जाते हैं।

## सामान्य समस्याएँ और समाधान
| समस्या | क्यों होता है | समाधान |
|-------|--------------|--------|
| **Exception: “X value … is out of valid range.”** | कोऑर्डिनेट प्रिसीजन ग्रिड के बाहर हैं। | `XOrigin`, `YOrigin`, या `XYScale` को समायोजित करें ताकि आपका डेटा शामिल हो, या इनपुट डेटा को परिभाषित रेंज के भीतर रखें। |
| **फ़ीचर GIS व्यूअर में नहीं दिख रहे** | लेयर सेव नहीं हुई या गलत स्पैशियल रेफ़रेंस। | सुनिश्चित करें `SpatialReferenceSystem.Wgs84` व्यूअर के CRS से मेल खाता है, और `Dataset.Create` सफल रहा है। |
| **M मान अनदेखे** | `MScale` 0 या बहुत कम सेट है। | उचित `MScale` (जैसे `1e4`) सेट करें ताकि माप मान संग्रहीत हो सकें। |

## अक्सर पूछे जाने वाले प्रश्न

**प्रश्न: क्या मैं Aspose.GIS for .NET को अन्य GIS फ़ाइल फ़ॉर्मेट्स के साथ उपयोग कर सकता हूँ?**  
उत्तर: हाँ, Aspose.GIS Shapefile, GeoJSON, KML, और कई अन्य फ़ॉर्मेट्स को सपोर्ट करता है।

**प्रश्न: क्या Aspose.GIS for .NET .NET Core के साथ संगत है?**  
उत्तर: बिल्कुल। यह लाइब्रेरी .NET Framework, .NET Core, और .NET 5/6+ के साथ काम करती है।

**प्रश्न: क्या मैं बफ़रिंग या इंटरसेक्शन जैसे स्पैशियल ऑपरेशन कर सकता हूँ?**  
उत्तर: हाँ, API में बफ़रिंग, इंटरसेक्टिंग, और दूरी गणना के मेथड शामिल हैं।

**प्रश्न: क्या Aspose.GIS कोऑर्डिनेट ट्रांसफ़ॉर्मेशन क्षमताएँ प्रदान करता है?**  
उत्तर: हाँ, आप बिल्ट‑इन रीप्रोज़िशन टूल्स का उपयोग करके विभिन्न स्पैशियल रेफ़रेंस सिस्टम के बीच ज्योमेट्री ट्रांसफ़ॉर्म कर सकते हैं।

**प्रश्न: क्या ट्रायल संस्करण उपलब्ध है?**  
उत्तर: हाँ, आप मुफ्त ट्रायल [वेबसाइट](https://releases.aspose.com/gis/net/) से डाउनलोड कर सकते हैं।
