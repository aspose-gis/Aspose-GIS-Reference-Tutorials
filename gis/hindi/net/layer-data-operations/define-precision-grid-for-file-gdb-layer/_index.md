# Aspose.GIS में फ़ाइल GDB लेयर के लिए ग्रिड कैसे सेट करें

## परिचय
इस ट्यूटोरियल में आप **फ़ाइल जियोडेटाबेस** ऑब्जेक्ट बनाएँगे और Aspose.GIS for .NET का उपयोग करके फ़ाइल जियोडेटाबेस (GDB) लेयर के लिए **ग्रिड सेट** करना सीखेंगे। एक प्रिसीजन ग्रिड को परिभाषित करने से आप **कोऑर्डिनेट रेंज को वैलिडेट** कर सकते हैं, आउट‑ऑफ़‑रेंज त्रुटियों को रोकते हैं, और यह सुनिश्चित करता है कि कोई भी **लेयर में फीचर जोड़ें** ऑपरेशन डेटा को सटीक रूप से स्टोर करे। हम हर चरण को विस्तार से बताएँगे, प्रत्येक सेटिंग के महत्व को समझाएँगे, और आपको **आउट ऑफ़ रेंज** परिस्थितियों को सहजता से **हैंडल** करने का तरीका दिखाएँगे।

## त्वरित उत्तर
- **“ग्रिड सेट” का क्या अर्थ है?** यह GIS लेयर के लिए कोऑर्डिनेट प्रिसीजन और वैध रेंज को परिभाषित करता है।  
- **प्रिसीजन ग्रिड क्यों उपयोग करें?** यह आपके डेटा को अवैध कोऑर्डिनेट्स से बचाता है और स्टोरेज दक्षता को बढ़ाता है।  
- **कौन सी लाइब्रेरी यह सुविधा प्रदान करती है?** Aspose.GIS for .NET.  
- **क्या मुझे लाइसेंस चाहिए?** एक ट्रायल उपलब्ध है; प्रोडक्शन के लिए एक कमर्शियल लाइसेंस आवश्यक है।  
- **क्या मैं इसे .NET Core के साथ उपयोग कर सकता हूँ?** हाँ, Aspose.GIS .NET Framework और .NET Core दोनों को सपोर्ट करता है।

## प्रिसीजन ग्रिड क्या है और इसे क्यों सेट करें?
एक प्रिसीजन ग्रिड पैरामीटरों (ऑरिजिन, स्केल आदि) का सेट है जो GIS इंजन को बताता है कि कोऑर्डिनेट मानों को कैसे राउंड और स्टोर किया जाए। ग्रिड को कॉन्फ़िगर करके आप **कोऑर्डिनेट रेंज को वैलिडेट** स्वचालित रूप से कर सकते हैं, और ग्रिड के बाहर कोई पॉइंट डालने का प्रयास करने पर एक्सेप्शन उत्पन्न होगा—जिससे आप विकास के शुरुआती चरण में ही **आउट ऑफ़ रेंज** परिस्थितियों को संभाल सकते हैं।

## फ़ाइल जियोडेटाबेस को प्रिसीजन ग्रिड के साथ क्यों बनाएं?
फ़ाइल जियोडेटाबेस बनाना आपको वेक्टर डेटा के लिए एक पोर्टेबल, हाई‑परफ़ॉर्मेंस कंटेनर देता है। निर्माण के समय प्रिसीजन ग्रिड जोड़ने से सुनिश्चित होता है:

- **सुसंगत डेटा गुणवत्ता** – प्रत्येक फीचर समान संख्यात्मक प्रिसीजन का पालन करता है।  
- **तेज़ इंडेक्सिंग** – इंजन कोऑर्डिनेट्स को अधिक कुशलता से स्टोर कर सकता है।  
- **प्रारंभिक त्रुटि पहचान** – आउट‑ऑफ़‑रेंज कोऑर्डिनेट्स को डेटासेट को भ्रष्ट करने से पहले पकड़ा जाता है।

## सामान्य उपयोग मामलों
- **फ़ील्ड डेटा संग्रह** जहाँ GPS डिवाइस थोड़ा बाहर के कोऑर्डिनेट्स उत्पन्न कर सकते हैं।  
- **डेटा माइग्रेशन** लेगेसी सिस्टम्स से जो विभिन्न कोऑर्डिनेट प्रिसीजन उपयोग करते थे।  
- **ऑटोमेटेड ETL पाइपलाइन्स** जिन्हें GIS डेटाबेस में डेटा लोड करने से पहले स्पेशियल इंटेग्रिटी लागू करनी होती है।

## नेमस्पेस इम्पोर्ट करें
पहले, Aspose.GIS के साथ काम करने के लिए आवश्यक नेमस्पेस इम्पोर्ट करें:

```csharp
using Aspose.Gis;
using Aspose.Gis.Formats.FileGdb;
using Aspose.Gis.Geometries;
using Aspose.Gis.SpatialReferencing;
using System;
using System.Text;
```

## फ़ाइल GDB लेयर में कोऑर्डिनेट ग्रिड कैसे कॉन्फ़िगर करें
नीचे एक स्टेप‑बाय‑स्टेप गाइड है जो दिखाता है कि ग्रिड को कैसे कॉन्फ़िगर करें, लेयर बनाएं, और सुरक्षित रूप से **लेयर में फीचर जोड़ें**।

### चरण 1: डेटासेट बनाएं
हम एक नया फ़ाइल जियोडेटाबेस डेटासेट बनाकर शुरू करते हैं। यही वह जगह है जहाँ लेयर रहेगा।

```csharp
var path = "Your Document Directory" + "PrecisionGrid_out.gdb";
using (var dataset = Dataset.Create(path, Drivers.FileGdb))
{
```

### चरण 2: प्रिसीजन ग्रिड विकल्प निर्धारित करें
यहाँ हम ग्रिड पैरामीटर निर्दिष्ट करते हैं। अपने प्रोजेक्ट के कोऑर्डिनेट सिस्टम से मेल खाने के लिए ऑरिजिन और स्केल को समायोजित करें।

```csharp
var options = new FileGdbOptions
{
    CoordinatePrecisionGrid = new FileGdbCoordinatePrecisionGrid
    {
        XOrigin = -400,
        YOrigin = -400,
        XYScale = 1e10,
        MOrigin = 0,
        MScale = 1e4,
    },
    EnsureValidCoordinatesRange = true,
};
```

*`EnsureValidCoordinatesRange = true` फ़्लैग Aspose.GIS को बताता है कि वह आप द्वारा जोड़े गए प्रत्येक फीचर के लिए **कोऑर्डिनेट रेंज को वैलिडेट** करे।*

### चरण 3: ग्रिड के साथ लेयर बनाएं
अब हम डेटासेट के अंदर एक नई लेयर बनाते हैं, जिसमें हमने अभी परिभाषित किए गए ग्रिड विकल्प लागू होते हैं। हम WGS84 स्पेशियल रेफ़रेंस सिस्टम का उपयोग करेंगे।

```csharp
using (var layer = dataset.CreateLayer("layer_name", options, SpatialReferenceSystem.Wgs84))
{
```

### चरण 4: लेयर में फीचर जोड़ें
हम दो पॉइंट फीचर बनाते हैं। पहला पॉइंट ग्रिड के भीतर है, जबकि दूसरा जानबूझकर ग्रिड के बाहर रखा गया है ताकि **आउट ऑफ़ रेंज** त्रुटियों को दर्शाया जा सके।

```csharp
var feature = layer.ConstructFeature();
feature.Geometry = new Point(10, 20) { M = 10.1282 };
layer.Add(feature);
feature = layer.ConstructFeature();
feature.Geometry = new Point(-410, 0) { M = 20.2343 };
```

### चरण 5: आउट‑ऑफ़‑रेंज फीचर जोड़ते समय एक्सेप्शन को हैंडल करें
दूसरे फीचर को जोड़ने का प्रयास करने पर एक्सेप्शन उत्पन्न होगा क्योंकि उसका X कोऑर्डिनेट (`-410`) परिभाषित ग्रिड के बाहर है। हम एक्सेप्शन को पकड़ते हैं और एक स्पष्ट संदेश आउटपुट करते हैं।

```csharp
try
{
    layer.Add(feature);
}
catch (GisException e)
{
    Console.WriteLine(e.Message); // X value -410 is out of valid range.
}
```

### चरण 6: सफ़ाई करें
`using` स्टेटमेंट्स स्वचालित रूप से डेटासेट और लेयर को बंद और डिस्पोज़ कर देते हैं, जिससे सभी रिसोर्सेज़ रिलीज़ हो जाते हैं।

## सामान्य समस्याएँ और समाधान
| समस्या | क्यों होता है | समाधान |
|-------|----------------|-----|
| **Exception: “X value … is out of valid range.”** | कोऑर्डिनेट्स प्रिसीजन ग्रिड के बाहर हैं। | अपने डेटा को शामिल करने के लिए `XOrigin`, `YOrigin`, या `XYScale` को समायोजित करें, या सुनिश्चित करें कि इनपुट डेटा परिभाषित रेंज के भीतर है। |
| **Features not appearing in GIS viewer** | लेयर सेव नहीं हुई या गलत स्पेशियल रेफ़रेंस। | सुनिश्चित करें कि `SpatialReferenceSystem.Wgs84` व्यूअर के CRS से मेल खाता है, और `Dataset.Create` सफल रहा है। |
| **M values ignored** | `MScale` 0 या बहुत कम सेट है। | माप मानों को स्टोर करने के लिए एक उचित `MScale` सेट करें (जैसे, `1e4`)। |

## समस्या निवारण टिप्स
- **ग्रिड एक्सटेंट्स को दोबारा जांचें** बड़े डेटा बैच लोड करने से पहले; `XOrigin` में छोटी टाइपो कई पंक्तियों को अस्वीकार कर सकती है।  
- **एक्सेप्शन संदेश को लॉग करें** (जैसे try‑catch ब्लॉक में दिखाया गया) फ़ाइल में जब ऑटोमेटेड इम्पोर्ट प्रोसेस कर रहे हों; इससे आउट‑ऑफ़‑रेंज डेटा में पैटर्न पहचानना आसान हो जाता है।  
- **`EnsureValidCoordinatesRange = false` केवल भरोसेमंद डेटा स्रोतों के लिए उपयोग करें** – इसे बंद करने से वैलिडेशन स्किप हो जाता है और इससे भ्रष्ट जियोमेट्री बन सकती है।  

## अक्सर पूछे जाने वाले प्रश्न

**प्रश्न: क्या मैं Aspose.GIS for .NET को अन्य GIS फ़ाइल फ़ॉर्मैट्स के साथ उपयोग कर सकता हूँ?**  
**उत्तर:** हाँ, Aspose.GIS Shapefile, GeoJSON, KML, और कई अन्य फ़ॉर्मैट्स को सपोर्ट करता है।

**प्रश्न: क्या Aspose.GIS for .NET .NET Core के साथ संगत है?**  
**उत्तर:** बिल्कुल। यह लाइब्रेरी .NET Framework, .NET Core, और .NET 5/6+ के साथ काम करती है।

**उत्तर:** हाँ, API में बफ़रिंग, इंटरसेक्टिंग, और दूरी गणना के मेथड शामिल हैं।

**प्रश्न: क्या Aspose.GIS कोऑर्डिनेट ट्रांसफ़ॉर्मेशन क्षमताएँ प्रदान करता है?**  
**उत्तर:** हाँ, आप बिल्ट‑इन रीप्रोज़िशन टूल्स का उपयोग करके विभिन्न स्पेशियल रेफ़रेंस सिस्टम्स के बीच जियोमेट्रीज़ को ट्रांसफ़ॉर्म कर सकते हैं।

**प्रश्न: क्या कोई ट्रायल संस्करण उपलब्ध है?**  
**उत्तर:** हाँ, आप [website](https://releases.aspose.com/gis/net/) से मुफ्त ट्रायल डाउनलोड कर सकते हैं।
