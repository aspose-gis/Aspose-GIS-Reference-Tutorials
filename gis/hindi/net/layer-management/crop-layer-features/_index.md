# लेयर फीचर को क्रॉप कैसे करें

## परिचय
इस ट्यूटोरियल में आप **लेयर को क्रॉप करने** का तरीका सीखेंगे, Aspose.GIS for .NET का उपयोग करके, जो आपको **पॉलीगॉन के साथ रास्टर को क्रॉप**, **रास्टर सेल साइज निकालने**, और **रास्टर की सीमा प्राप्त करने** की सुविधा देता है, जिससे सटीक जियोस्पेशियल विश्लेषण संभव हो जाता है। चाहे आप वेब मैप के लिए डेटा तैयार कर रहे हों, सैटेलाइट इमेज को ट्रिम कर रहे हों, या किसी रुचि के क्षेत्र को अलग कर रहे हों, यह चरण‑दर‑चरण गाइड आपको बिल्कुल सही तरीका दिखाएगा।

## त्वरित उत्तर
- **“क्रॉप लेयर” का क्या अर्थ है?** यह एक रास्टर या वेक्टर लेयर को प्रदान किए गए ज्योमेट्री की सीमाओं तक सीमित करता है।  
- **इस उदाहरण में कौन सी ज्योमेट्री उपयोग की गई है?** WKT फ़ॉर्मेट में परिभाषित एक पॉलीगॉन।  
- **क्या क्रॉप करने के बाद सेल साइज निकाली जा सकती है?** हाँ – `CellSize` प्रॉपर्टी वह जानकारी देती है।  
- **कोड चलाने के लिए लाइसेंस चाहिए?** मूल्यांकन के लिए एक टेम्पररी लाइसेंस काम करता है; प्रोडक्शन के लिए पूर्ण लाइसेंस आवश्यक है।  
- **कौन से .NET संस्करण समर्थित हैं?** .NET Framework 4.5+, .NET Core 3.1+, .NET 5/6/7।

## “लेयर को क्रॉप कैसे करें” क्या है?
लेयर को क्रॉप करना मतलब डेटासेट को एक विशिष्ट भौगोलिक क्षेत्र तक सीमित करना है, जिससे परिभाषित आकार के बाहर का सब कुछ हटाया जाता है। इससे फ़ाइल आकार कम होता है, प्रोसेसिंग तेज़ होती है, और विश्लेषण उस क्षेत्र पर केंद्रित रहता है जिसमें आपकी रुचि है।

## Aspose.GIS से क्रॉप करने के फायदे
- **उच्च‑प्रदर्शन रास्टर हैंडलिंग** – बड़े GeoTiff फ़ाइलों के लिए आदर्श।  
- **सरल API** – किसी भी ज्योमेट्री के साथ एक‑लाइन `Crop` कॉल।  
- **पूर्ण .NET संगतता** – डेस्कटॉप, सर्वर, और क्लाउड एप्लिकेशन में काम करता है।  
- **सटीक स्पैशियल रेफ़रेंस हैंडलिंग** – CRS जानकारी को स्वचालित रूप से संरक्षित करता है।

## पूर्वापेक्षाएँ
जियोस्पेशियल मैनिपुलेशन की जादू में डुबने से पहले सुनिश्चित करें कि आपके पास निम्नलिखित चीज़ें उपलब्ध हों:

- Aspose.GIS for .NET लाइब्रेरी: सुनिश्चित करें कि आपने अपने .NET प्रोजेक्ट में Aspose.GIS लाइब्रेरी इंस्टॉल की हुई है। आप इसे [यहाँ](https://releases.aspose.com/gis/net/) से डाउनलोड कर सकते हैं।  
- डॉक्यूमेंट डायरेक्टरी: अपने दस्तावेज़ों को संग्रहीत करने के लिए एक फ़ोल्डर बनाएं। प्रदान किए गए कोड में `"Your Document Directory"` को अपने वास्तविक डॉक्यूमेंट डायरेक्टरी पाथ से बदलें।

अब, चलिए चरण‑दर‑चरण गाइड में डुबकी लगाते हैं।

## नेमस्पेस इम्पोर्ट करें
Aspose.GIS की पूरी शक्ति का उपयोग करने के लिए आवश्यक नेमस्पेस इम्पोर्ट करें:

```csharp
using System;
using System.IO;
using Aspose.Gis;
using Aspose.Gis.Geometries;
```

## चरण 1: लेयर खोलें और क्रॉप करें (पॉलीगॉन के साथ रास्टर को क्रॉप करें)
GeoTiff लेयर को खोलें और परिभाषित पॉलीगॉन के आधार पर उसे क्रॉप करें। यह सुनिश्चित करता है कि आपका जियोस्पेशियल डेटा एक विशिष्ट रुचि क्षेत्र तक सीमित हो।

```csharp
using (var layer = Drivers.GeoTiff.OpenLayer(Path.Combine(filesPath, "geodetic_world.tif")))
using (var warped = layer.Crop(Geometry.FromText("POLYGON ((-160 0, 0 60, 160 0, 0 -160, -160 0))")))
{
```

## चरण 2: रास्टर जानकारी प्राप्त करें (रास्टर सेल साइज निकालें और रास्टर की सीमा प्राप्त करें)
लेयर क्रॉप हो जाने के बाद, रास्टर डेटा के महत्वपूर्ण विवरण निकालें, जैसे सेल साइज, स्पैशियल रेफ़रेंस सिस्टम, और बाउंड्स।

```csharp
// read and print raster
var cellSize = warped.CellSize;
var extent = warped.GetExtent();
var spatialRefSys = warped.SpatialReferenceSystem;
var code = spatialRefSys == null ? "'no srs'" : spatialRefSys.EpsgCode.ToString();
var bounds = warped.Bounds;
```

## चरण 3: जानकारी प्रदर्शित करें
निकाली गई जानकारी को प्रिंट करें ताकि आप समझ सकें कि क्रॉपिंग प्रक्रिया ने आपके जियोस्पेशियल डेटा को कैसे प्रभावित किया।

```csharp
Console.WriteLine($"cellSize: {cellSize}");
Console.WriteLine($"source extent: {layer.GetExtent()}");
Console.WriteLine($"target extent: {extent}");
Console.WriteLine($"spatialRefSys: {code}");
Console.WriteLine($"bounds: {bounds}");
```

आवश्यकतानुसार इन चरणों को दोहराएँ ताकि आपका जियोस्पेशियल डेटा विशिष्ट प्रोजेक्ट आवश्यकताओं के अनुसार परिष्कृत और अनुकूलित हो सके।

## सामान्य समस्याएँ और समाधान
| समस्या | समाधान |
|-------|----------|
| **पॉलीगॉन की दिशा गलत है** | सुनिश्चित करें कि WKT पॉलीगॉन राइट‑हैंड रूल (बाहरी रिंग के लिए काउंटर‑क्लॉकवाइज़) का पालन करता हो। |
| **स्पैशियल रेफ़रेंस गायब है** | जाँचें कि स्रोत GeoTiff में CRS मौजूद है; यदि नहीं, तो क्रॉप करने से पहले मैन्युअली सेट करें। |
| **बड़े रास्टर पर प्रदर्शन धीमा हो रहा है** | `layer.Crop` को डाउन‑सैंपल्ड कॉपी पर उपयोग करें या रास्टर को टाइल्स में प्रोसेस करें। |
