# Aspose.GIS for .NET के साथ SVG मानचित्र कैसे जनरेट करें और शहर जोड़ें

## परिचय
Aspose.GIS for .NET की रोमांचक दुनिया में आपका स्वागत है! इस चरण‑दर‑चरण मार्गदर्शिका में, आप **मानचित्र में शहर जोड़ने** और **SVG मानचित्र** आउटपुट जनरेट करने के बारे में सीखेंगे जो किसी भी स्क्रीन पर शानदार दिखता है। चाहे आप डेस्कटॉप डैशबोर्ड, वेब‑आधारित GIS पोर्टल, या प्रिंटेबल रिपोर्ट बना रहे हों, वही कोड .NET Framework, .NET Core, और .NET 5/6 पर काम करता है। चलिए पूरी प्रक्रिया को देखते हैं—मानचित्र के आयाम सेट करने से लेकर एक साफ़, स्केलेबल SVG फ़ाइल निर्यात करने तक।

## त्वरित उत्तर
- **इस ट्यूटोरियल में क्या कवर किया गया है?** मानचित्र पर शहर बिंदु जोड़ना और परिणाम को SVG फ़ाइल के रूप में निर्यात करना।  
- **कौन सी लाइब्रेरी आवश्यक है?** Aspose.GIS for .NET (नि:शुल्क ट्रायल उपलब्ध)।  
- **क्या मुझे लाइसेंस चाहिए?** विकास के लिए ट्रायल काम करता है; उत्पादन उपयोग के लिए एक व्यावसायिक लाइसेंस आवश्यक है।  
- **क्या मैं इसे वेब ऐप्स में उपयोग कर सकता हूँ?** बिल्कुल – वही कोड ASP.NET, Blazor, और अन्य .NET वेब फ्रेमवर्क में चलता है।  
- **कौन सा आउटपुट फ़ॉर्मेट उत्पन्न होता है?** एक उच्च‑रिज़ॉल्यूशन SVG मानचित्र जो ब्राउज़र तुरंत रेंडर करते हैं और वेक्टर एडिटर आगे संपादित कर सकते हैं।

## “generate SVG map” क्या है?
*Generating an SVG map* का अर्थ है भौगोलिक डेटा को Scalable Vector Graphics फ़ाइल में बदलना जो ज्यामिति, शैलियों और इंटरैक्टिविटी को बिना इमेज को रास्टर किए संरक्षित रखती है। SVG फ़ाइलें किसी भी ज़ूम स्तर पर स्पष्ट रहती हैं और वेब विज़ुअलाइज़ेशन के लिए आदर्श हैं।

## Aspose.GIS के साथ SVG मानचित्र क्यों जनरेट करें?
Aspose.GIS **70+ इनपुट और आउटपुट फ़ॉर्मेट** (Shapefile, GeoJSON, KML, और GML सहित) को सपोर्ट करता है और **सब‑पिक्सेल सटीकता** के साथ मानचित्र रेंडर कर सकता है जबकि मेमोरी उपयोग कम रखता है। लाइब्रेरी **सैकड़ों‑पृष्ठ डेटा सेट** को पूरी फ़ाइल को मेमोरी में लोड किए बिना प्रोसेस करती है, जिससे यह बड़े‑पैमाने के GIS प्रोजेक्ट्स के लिए उपयुक्त बनती है।

## मानचित्र आयाम कैसे सेट करें?
`Map` वह कोर क्लास है जो रेंडरिंग सतह को रखती है और लेयर्स को प्रबंधित करती है।  
पहले अपने मानचित्र कैनवास का आकार सेट करें ताकि रेंडरर को पता हो कि कितना स्थान आवंटित करना है। आप एक `Map` ऑब्जेक्ट बनाते हैं, पिक्सेल में चौड़ाई और ऊँचाई पास करते हैं, और वैकल्पिक रूप से बैकग्राउंड रंग निर्धारित कर सकते हैं। यह चरण अंतिम SVG के आस्पेक्ट रेशियो, फ़ाइल आकार, और विभिन्न डिवाइसों पर दृश्य स्पष्टता को नियंत्रित करता है।

```csharp
string dataDir = "Your Document Directory";
using (var map = new Map(800, 476))
{
    // Additional code for map setup can be added here.
}
```

## बेस मानचित्र के लिए वेक्टर लेयर कैसे ड्रॉ करें?
`DrawVectorLayer` एक दिए गए सिम्बोलाइज़र का उपयोग करके मानचित्र पर वेक्टर लेयर रेंडर करता है।  
`Map` क्लास एक `DrawVectorLayer` मेथड प्रदान करती है जो shapefile पढ़ती है और प्रत्येक फीचर को `SimpleFill` शैली का उपयोग करके रेंडर करती है। आप फ़िल रंग, आउटलाइन मोटाई, और अपारदर्शिता को अपने विज़ुअल थीम के अनुसार कस्टमाइज़ कर सकते हैं, और समृद्ध कार्टोग्राफिक प्रभावों के लिए ट्रांसपैरेंसी या पैटर्न फ़िल भी लागू कर सकते हैं।

```csharp
var baseMapSymbolizer = new SimpleFill { FillColor = Color.Salmon, StrokeWidth = 0.75 };
map.Add(VectorLayer.Open(dataDir + "basemap.shp", Drivers.Shapefile), baseMapSymbolizer);
```

## GeoJSON कैसे लोड करें और मानचित्र में शहर जोड़ें?
`FeatureBasedConfiguration` एक डेलीगेट है जो आपको प्रत्येक फीचर को उसके एट्रिब्यूट्स के आधार पर स्टाइल करने देता है।  
GeoJSON फ़ाइल लोड करने से आपको शहरों का प्रतिनिधित्व करने वाले पॉइंट फीचर्स का संग्रह मिलता है। `FeatureBasedConfiguration` डेलीगेट पास करके आप प्रत्येक शहर मार्कर को जनसंख्या या क्षेत्र जैसे एट्रिब्यूट्स के आधार पर स्टाइल कर सकते हैं। आप अतिरिक्त डेटा डाइमेंशन को दर्शाने के लिए फीचर्स को फ़िल्टर भी कर सकते हैं या सिम्बल आकार को डायनामिक रूप से समायोजित कर सकते हैं।

```csharp
var citiesSymbolizer = new SimpleMarker() { FillColor = Color.LightBlue };
citiesSymbolizer.FeatureBasedConfiguration = (feature, symbolizer) =>
{
    // Additional configuration logic can be added here.
};
map.Add(VectorLayer.Open(dataDir + "points.geojson", Drivers.GeoJson), citiesSymbolizer);
```

## मानचित्र को SVG फ़ाइल के रूप में कैसे निर्यात करें?
`Map.Save` रेंडर किए गए मानचित्र को निर्दिष्ट फ़ॉर्मेट में फ़ाइल में आउटपुट करता है।  
`SaveFormat.Svg` आर्ग्यूमेंट के साथ `Map.Save` को कॉल करने से रेंडर किया गया वेक्टर डेटा डिस्क पर एक SVG फ़ाइल में लिखा जाता है। परिणामी `cities_out.svg` को सीधे किसी भी आधुनिक ब्राउज़र में खोला जा सकता है या Inkscape जैसे टूल्स से संपादित किया जा सकता है। आप आगे की स्टाइलिंग के लिए DPI निर्दिष्ट कर सकते हैं या CSS एम्बेड कर सकते हैं।

```csharp
map.Render(dataDir + "cities_out.svg", Renderers.Svg);
```

## सामान्य समस्याएँ और समाधान
- **Missing data file error** – सुनिश्चित करें कि आपके shapefile और GeoJSON के रिलेटिव पाथ सही हैं; डिबगिंग के दौरान एब्सोल्यूट पाथ का उपयोग करें।  
- **Cities not visible** – सुनिश्चित करें कि GeoJSON का कोऑर्डिनेट रेफ़रेंस सिस्टम बेस मानचित्र के CRS से मेल खाता है, या `Geometry.Transform` का उपयोग करके डेटा को पुनः‑प्रोजेक्ट करें।  
- **SVG appears blank** – जाँचें कि मानचित्र की बैकग्राउंड रंग पूरी तरह से ट्रांसपेरेंट नहीं है; रेंडरिंग की पुष्टि के लिए हल्का फ़िल सेट करें।
