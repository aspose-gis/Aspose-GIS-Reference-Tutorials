# पॉइंट इनसाइड पॉलीगॉन C# – जियोमेट्री कंटेन्स एनीदर की जाँच

## परिचय
यदि आप **geospatial analysis .net** प्रोजेक्ट्स पर काम कर रहे हैं, तो सबसे सामान्य कार्यों में से एक यह निर्धारित करना है कि कोई विशिष्ट स्थान (एक पॉइंट) परिभाषित क्षेत्र (एक पॉलीगॉन) के भीतर आता है या नहीं। इस ट्यूटोरियल में हम आपको चरण‑दर‑चरण दिखाएंगे कि **point inside polygon c#** जाँच को **Aspose.GIS .NET** लाइब्रेरी का उपयोग करके कैसे किया जाए। चाहे आप मैपिंग एप्लिकेशन, लोकेशन‑बेस्ड सर्विस, या कोई भी समाधान बना रहे हों जिसे स्पैशियल कंटेन्स लॉजिक की आवश्यकता हो, नीचे दिए गए कोड स्निपेट्स आपको कुछ ही मिनटों में शुरू करने में मदद करेंगे।

## त्वरित उत्तर
- **“point inside polygon c#” का क्या अर्थ है?** यह एक स्पैशियल क्वेरी है जो तब `true` लौटाती है जब एक पॉइंट जियोमेट्री पूरी तरह से एक पॉलीगॉन जियोमेट्री के भीतर स्थित हो।  
- **.NET में यह कौन सी लाइब्रेरी संभालती है?** Aspose.GIS for .NET `SpatiallyContains` और `Within` मेथड्स प्रदान करती है।  
- **क्या मुझे लाइसेंस चाहिए?** एक फ्री ट्रायल उपलब्ध है; प्रोडक्शन उपयोग के लिए कमर्शियल लाइसेंस आवश्यक है।  
- **क्या यह .NET Core / .NET 6+ के साथ संगत है?** हाँ – Aspose.GIS आधुनिक .NET रनटाइम्स को पूरी तरह सपोर्ट करती है।  
- **इम्प्लीमेंटेशन में कितना समय लगेगा?** कोड कॉपी करके उदाहरण चलाने में लगभग 10 मिनट लगेंगे।

## point inside polygon c# क्या है?
*पॉइंट इनसाइड पॉलीगॉन* टेस्ट यह जांचता है कि `Point` ऑब्जेक्ट के निर्देशांक `Polygon` ऑब्जेक्ट की सीमाओं के भीतर स्थित हैं या नहीं। C# में यह आमतौर पर जियोमेट्री लाइब्रेरीज़ के माध्यम से किया जाता है जो **Ray Casting** या **Winding Number** एल्गोरिदम को लागू करती हैं। Aspose.GIS इन विवरणों को एब्स्ट्रैक्ट करके एक सरल API प्रदान करती है: `polygon.SpatiallyContains(point)`।

- **उच्च‑प्रदर्शन स्पैशियल ऑपरेशन्स** – बड़े डेटासेट्स के लिए ऑप्टिमाइज़्ड।  
- **क्रॉस‑प्लेटफ़ॉर्म** – .NET Framework, .NET Core, और .NET 5/6+ पर काम करता है।  
- **व्यापक दस्तावेज़ीकरण** – geospatial analysis .net परिदृश्यों के लिए कई उदाहरण उपलब्ध हैं।

## नेमस्पेसेस इम्पोर्ट करें
अपने C# प्रोजेक्ट में Aspose.GIS की कार्यक्षमताओं को उपयोग करने के लिए आवश्यक नेमस्पेसेस इम्पोर्ट करें:
```csharp
using Aspose.Gis.Geometries;
using System;
using System.Collections.Generic;
using System.Linq;
using System.Text;
using System.Threading.Tasks;
```

## चरण 1: जियोमेट्री ऑब्जेक्ट्स परिभाषित करें
पहले, Aspose.GIS क्लासेज़ का उपयोग करके जियोमेट्री ऑब्जेक्ट्स परिभाषित करें। यहाँ हम एक बाहरी रिंग और एक आंतरिक रिंग (एक होल) वाले पॉलीगॉन को बनाते हैं, फिर एक पॉइंट बनाते हैं जिसे हम कंटेन्स के लिए टेस्ट करेंगे।
```csharp
var geometry1 = new Polygon();
geometry1.ExteriorRing = new LinearRing(new[]
{
    new Point(0, 0),
    new Point(0, 4),
    new Point(4, 4),
    new Point(4, 0),
    new Point(0, 0),
});
geometry1.AddInteriorRing(new LinearRing(new[]
{
    new Point(1, 1),
    new Point(1, 3),
    new Point(3, 3),
    new Point(3, 1),
    new Point(1, 1),
}));
var geometry2 = new Point(2, 2);
```

## चरण 2: स्पैशियल कंटेन्स जाँचें
अब जाँचें कि पॉलीगॉन **geometry1** पॉइंट **geometry2** को कंटेन करता है या नहीं। `SpatiallyContains` मेथड `false` लौटाता है क्योंकि पॉइंट आंतरिक रिंग (होल) के भीतर स्थित है।
```csharp
Console.WriteLine(geometry1.SpatiallyContains(geometry2)); // False
```

## चरण 3: दूसरा जियोमेट्री परिभाषित करें
अब हम एक दूसरा पॉइंट परिभाषित करते हैं जो बाहरी रिंग में है लेकिन आंतरिक रिंग के बाहर स्थित है।
```csharp
var geometry3 = new Point(0.5, 0.5);
```

## चरण 4: फिर से स्पैशियल कंटेन्स जाँचें
नए पॉइंट के साथ वही कंटेन्स जाँच चलाने पर `true` मिलता है, जिससे पुष्टि होती है कि पॉइंट वास्तव में पॉलीगॉन की बाहरी सीमा के भीतर है।
```csharp
Console.WriteLine(geometry1.SpatiallyContains(geometry3)); // True
```

## चरण 5: समकक्ष कार्यक्षमता
Aspose.GIS इनवर्स मेथड `Within` भी प्रदान करती है। निम्न पंक्ति दर्शाती है कि `geometry3.Within(geometry1)` वही परिणाम देता है जैसा `geometry1.SpatiallyContains(geometry3)` देता है।
```csharp
Console.WriteLine(geometry3.Within(geometry1)); // True
```

## सामान्य समस्याएँ और समाधान
| समस्या | क्यों होता है | समाधान |
|-------|----------------|-----|
| **अनपेक्षित `false` परिणाम** | पॉइंट पॉलीगॉन के होल (आंतरिक रिंग) के भीतर स्थित है। | सही पॉलीगॉन के विरुद्ध टेस्ट करें या साधारण पॉलीगॉन के बिना होल के लिए `geometry1.ExteriorRing` का उपयोग करें। |
| **NullReferenceException** | `SpatiallyContains` कॉल करने से पहले जियोमेट्री ऑब्जेक्ट्स इनिशियलाइज़ नहीं किए गए। | स्पैशियल मेथड्स को कॉल करने से पहले दोनों पॉलीगॉन और पॉइंट ऑब्जेक्ट्स को इंस्टैंशिएट करें। |
| **बड़े डेटासेट्स पर प्रदर्शन में गिरावट** | लूप्स के भीतर बार‑बार जियोमेट्री ऑब्जेक्ट्स बनाना। | जियोमेट्री इंस्टेंस को री‑यूज़ करें या `GeometryCollection` का उपयोग करके बैच प्रोसेसिंग करें। |
